# 01_AI_Environment_Final

AI環境整備ロードマップ（確定版）

---

## 0. 全体像

- Input：Typeless / Notta / Grok / Gemini
- Process：Cursor / Claudeプロジェクト / NotebookLM
- Store：Obsidian
- Output：Claude Code

---

## 1. ツール役割定義（固定）

- Typeless：音声思考の高速入力
- Notta：会議の文字起こし
- Grok：リアルタイム情報収集
- Gemini：Google系連携用途
- Claudeプロジェクト：戦略壁打ち・文脈蓄積
- Cursor：全操作の中核UI・最終出力
- NotebookLM：ソース限定の理解・RAG
- Obsidian：知識資産の長期蓄積
- Claude Code：業務フロー化・自動化

---

## 2. 使い分け判断

- リアルタイム情報：Grok
- ソース準拠で深掘り：NotebookLM
- 文脈込みの戦略壁打ち：Claudeプロジェクト
- 生成・編集・仕上げ：Cursor
- 自動化実装：Claude Code

---

## 3. 情報フロー原則

1. 収集
2. 理解
3. 構造化
4. 実行
5. 蓄積

補足：Obsidianには「生ログ」ではなく「資産化済みアウトプット」を保存する。

---

## 4. NotebookLM運用

- 8冊構成で領域分離（Career / Business / RINGBELL / Finance / Athlete / Fashion / Journal / AI活用）
- NotebookLMで抽出した洞察のみをObsidianへ転記
- 「答えられない質問」を記録し、ソース追加で改善

---

## 5. Android運用

- 移動中：Typeless / Grok / Claudeアプリ
- 週次：Inbox整理、Journal分析、Obsidian反映

---

## 6. 運用品質ルール

- 1ツール1責任（役割を混ぜない）
- 生成物は利用前に事実・文脈を確認
- ツール停止時は代替経路を準備（例：Notta停止時は標準書き起こし）
