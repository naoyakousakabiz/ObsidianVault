# AI戦略 統合ドキュメント

本ドキュメントは、AI活用を「導入」ではなく「運用可能な業務OS」として設計し、実行し、改善するための単一運用版。

## 目的

- 人間は判断に集中し、AIは処理を担う構造を作る
- 個人の成果を再現可能な業務フローへ変換する
- 実績を定量で示し、対外説明できる状態を維持する

## 基本原則

1. 1ツール1責任（役割を混ぜない）
2. 情報は最適レイヤーで処理する
3. 生成物は資産化して再利用する
4. 品質とガバナンスを先に定義してから自動化する

## 運用アーキテクチャ

### ツールレイヤー

- Input：Typeless / Notta / Grok / Gemini
- Process：Cursor / Claudeプロジェクト / NotebookLM
- Store：Obsidian
- Output：Claude Code

### 役割定義

- Typeless：音声思考の高速入力
- Notta：会議の文字起こし
- Grok：リアルタイム情報収集
- Gemini：Google連携系タスク
- Claudeプロジェクト：文脈込みの戦略壁打ち
- NotebookLM：ソース限定の理解・検証
- Cursor：編集・統合・最終出力
- Claude Code：業務フロー化・自動化
- Obsidian：資産化済み知識の長期蓄積

### 標準フロー（全業務共通）

1. 収集
2. 理解
3. 構造化
4. 実行
5. 蓄積

運用ルール：Obsidianには生ログを入れず、要点化・再利用可能化した資産のみを保存する。

## データガバナンス

### データ区分

- レベルA（公開可）：一般情報・公開前提草稿
- レベルB（社内限定）：社内資料・運用メモ
- レベルC（機微情報）：個人情報・顧客情報・契約情報

### 投入ルール

- A：外部AI投入可
- B：匿名化前提で投入可否を判断
- C：原則投入不可（必要時は匿名化+責任者承認）

### 保存ルール

- 生データは保存期間を事前定義（例：90日）
- 顧客向け最終文面は人間承認後に送信
- 監査に必要な判断ログは残す

## 品質ゲート

- Gate1 事実整合：日付・数値・固有名詞の照合
- Gate2 文脈整合：顧客背景・過去経緯との矛盾確認
- Gate3 トーン整合：対外/社内の文体基準確認
- Gate4 リスク整合：機微情報・誤送信リスク確認

判定基準：
- Green：即利用可
- Yellow：修正後利用
- Red：入力/プロンプトを修正して再生成

## KPI設計（3層）

### 活動KPI

- 自動化フロー実行回数/週
- 削減時間（投入時間に対する回収）
- ドキュメント更新回数

### 品質KPI

- 初稿採用率
- 人手修正率
- 差し戻し件数

### 事業KPI

- 提案作成リードタイム
- 受注率/継続率
- 粗利寄与

### フェーズ詳細（実行項目・成果物・完了条件）

#### Phase1（業務自動化）

- 実行項目
  - 面談準備、議事録整理、返信草案など高頻度業務を優先実装
  - 使うたびに「良かった点/改善点/次アクション」を記録
- 成果物
  - 自動化フロー一覧
  - 週次の使用ログと削減時間ログ
- 完了条件
  - 自動化フロー5件以上が週3回以上の実務で稼働
  - 手作業比で時間短縮を確認（定量）

#### Phase2（技術スタック構築）

- 実行項目
  - `CLAUDE.md` の運用規約化
  - Skills 3本を優先作成（meeting-summary / email-reply / slack-report）
  - MCPを1本連携し、業務フローに組み込む
- 成果物
  - 公開可能なリポジトリ
  - ライブデモ手順書
- 完了条件
  - `CLAUDE.md` / Skills / MCPのデモ実施が可能
  - 設計意図を説明できる

#### Phase3（コンサル設計）

- 実行項目
  - 外部1社以上にヒアリング
  - 業務棚卸し -> 優先順位付け -> PoC提案を実施
- 成果物
  - AI活用度診断シート
  - 提案資料とケーススタディ
- 完了条件
  - 1社以上の診断/提案が事例化済み
  - 「ヒアリング->提案->実装計画」まで一気通貫で説明可能

#### Phase4（自分用AIツール）

- 実行項目
  - 日常で使う小ツールを5本開発
  - 利用頻度と改善ログを定着化
- 成果物
  - 個人運用ツール群
  - 運用マニュアル（短いもので可）
- 完了条件
  - 日常利用AIツール5件が継続稼働

#### Phase5（AIアプリ開発）

- 実行項目
  - 1テーマを選びWebアプリとして実装
  - API連携/RAG/基本的な運用導線を実装
- 成果物
  - デモ可能なWebアプリ
  - 利用シナリオと検証メモ
- 完了条件
  - デモ可能なWebアプリ1本以上

#### Phase6（AIレクチャー）

- 実行項目
  - 社内勉強会または小規模ワークショップを実施
  - 受講者向けガイドを作成
- 成果物
  - 研修スライド
  - ハンズオン資料
- 完了条件
  - 3名以上への導入支援実績
  - 研修資料が再利用可能な状態

#### Phase7（ナレッジ化）

- 実行項目
  - Phase1-6の成功/失敗パターンを整理
  - FAQとトラブルシュートを蓄積
- 成果物
  - 導入事例集
  - 業務自動化パターン集
  - FAQ集
- 完了条件
  - 事例集/パターン集/FAQの体系化が完了

#### Phase8（新規事業検討）

- 実行項目
  - 収益化モデル（コンサル/ツール/教育）を比較
  - 既存業務との優先順位を判断
- 成果物
  - 事業選択メモ
  - 検証計画（PoC計画）
- 完了条件
  - 事業化のGo/No-Goを判断できる状態
