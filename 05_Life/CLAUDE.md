# 05_Life コンテキスト

## 役割
世帯の生活インフラ（健康・家計・契約・イベント・合意・保険）の正本フォルダ。

## フォルダ構成

| フォルダ | 役割 |
|---|---|
| `11_health/` | 健康管理（健診・ワクチン・測定）二人分 |
| `20_events/` | ライフイベント（期限あり。年別フォルダで管理） |
| `30_finance/` | 家計・FP・サブスク |
| `40_agreements/` | 継続する合意・ルール・方針 |
| `50_AI自動化/` | ライフOS自動化の設計・実装・ログ |
| `60_insurance/` | 保険方針・契約状況 |

将来候補（未作成）: `70_decisions/`（重要意思決定ログ）、`80_documents/`（書類在り処メモ）

## フォルダ判断基準
- カレンダーに締切が立つ → `20_events/`
- 継続的ルール・合意 → `40_agreements/`
- お金の数字・契約 → `30_finance/`
- 健康・医療 → `11_health/`
- 保険 → `60_insurance/`

## AI運用ルール
- ハブ（年次まとめ・各README）を先に更新し、詳細は個別ファイルへ
- 「事実（日付・数値・担当）」「判断理由」「次のアクション日」を分けて記載
- 個人を特定しやすい情報（口座番号・証券番号・保険番号）はVaultに書かない
- 今後入力すべきデータは `00_AI活用TODO.md` に先にメモしてから実装
- `20_events/` は年フォルダ単位で保持。完了後も同年フォルダ内で `状態: 完了` を維持

## ファイル命名
- イベント系: `YYYYMMDD_内容.md`（未確定は `TBD_内容.md`）
- 継続管理系: `NN_内容.md`
- `TBD_` が日付確定したら `YYYYMMDD_` にリネーム
