## 3. P1-1 情報収集（`p1-1-collect.py`）

- **スクリプト**: `05_Life/50_AI自動化/scripts/p1-1-collect.py`
- **収集ソース**:
  - AIニュース: Google News RSS（キーワード: AI）
  - Web記事: `20_Input/03_RSS/_購読リスト.md` に定義した RSS フィード
  - YouTube: 登録チャンネルの最新動画
  - X: 保留（`X_BEARER_TOKEN` 設定後に有効化）
- **重複除外**: `20_Input/05_日次情報収集/prev_collect.md` で前回 URL と照合
- **表示上限**: Web記事10件 / YouTube 全件 / AIニュース5件

## 4. P1-2 朝ブリーフ（`morning_briefing.sh`）

- **スクリプト**: `05_Life/50_AI自動化/scripts/morning_briefing.sh`
- **配信時刻**: 毎朝 06:00 JST
- **処理フロー**:
  1. `gog calendar events --today` で Google Calendar の当日予定を取得
  2. Daily ノート（`80_Journal/01_Daily/YYYY-MM-DD.md`）の「今日やること」セクションを awk で抽出
  3. Slack Block Kit で組み立てて `chat.postMessage` で送信

- **gogcli インストール（CI）**: GitHub Releases から Linux バイナリを直接取得  
  `https://github.com/steipete/gogcli/releases/download/v0.12.0/gogcli_0.12.0_linux_amd64.tar.gz`
- **credentials パス（Linux）**: `~/.config/gogcli/credentials.json`
- **トークン期限切れ検知**: カレンダー取得が空の場合、予定欄に警告を表示。詳細は `90_運用ログ/GOG_TOKEN_更新手順.md`

## 5. P1-3 リマインダー（`life-reminder.sh`）

- **スクリプト**: `05_Life/50_AI自動化/scripts/life-reminder.sh <type>`
- **呼び出し**: `bash life-reminder.sh breakfast` のように `type` を引数で渡す

### 5.1 スケジュール済み（GitHub Actions）

| 時刻 (JST) | UTC cron | type | ワークフロー判定 |
|:--|:--|:--|:--|
| 06:00 | `0 21 * * *` | `training` | HOUR=21 |
| 07:30 | `30 22 * * *` | `breakfast` | HOUR=22 |
| 12:00 | `0 3 * * *` | `lunch` | HOUR=03 |
| 15:30 | `30 6 * * *` | `snack` | HOUR=06 |
| 19:00 | `0 10 * * *` | `dinner` | HOUR=10 |

### 5.2 追加種別（手動 dispatch または今後スケジュール追加可）

| type | 用途 |
|---|---|
| `work_end` | 仕事終了・ジムへの切り替えトリガ |
| `voice_journal` | 音声ジャーナルの録音促進 |
| `bedtime` | 就寝トリガ |

### 5.3 コンテンツ正本

| 種別 | 追加コンテンツ | 正本ファイル |
|---|---|---|
| breakfast / lunch / snack / dinner | 栄養ポイント | `09_Athlete/70_食事/食事リマインド本文.md`（`### type` セクション） |
| training | 週間練習メニュー | `09_Athlete/10_トライアスロン/週間練習メニュー.md` |
| その他 | なし | — |

## 6. テスト手順

テンプレートや配信ロジックを変更したら以下を実施する。

```bash
gh workflow run morning-briefing.yml
gh workflow run p1-1-collect.yml
gh workflow run life-reminder.yml -f type=breakfast
```

確認観点:
- Bot 送信であること（アプリ名が表示される）
- `@mention` が正しいこと
- 改行・装飾・URL リンクが崩れていないこと

---

## 7. 障害時の対応

| 症状 | 確認先 | 対処 |
|---|---|---|
| 予定が「⚠️ 取得失敗」と表示 | GOG_TOKEN_B64 の期限切れ | `90_運用ログ/GOG_TOKEN_更新手順.md` |
| Slack に届かない | `SLACK_BOT_TOKEN` / チャンネル ID | GitHub Secrets を確認 |
| p1-1 が収集0件 | RSS フィードの URL 変更 | `20_Input/03_RSS/_購読リスト.md` を確認 |
| 重複通知が発生 | ローカル launchd が復活 | `launchctl list \| grep com.lifeos` で確認・停止 |
