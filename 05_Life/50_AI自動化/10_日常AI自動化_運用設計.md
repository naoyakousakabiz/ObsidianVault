## 1. 目的

「Slack を開けば次の行動が決まる」状態を毎日維持する。

- 情報収集・予定確認・行動トリガはすべて自動配信
- 記録は Obsidian、即時行動は Slack という役割分担を維持
- Mac の電源・スリープに依存しない **クラウド本番** を基本とする

---

## 2. 運用原則

1. **単一責務** — 1ジョブ = 1チャンネル・1目的
2. **重複禁止** — 同じジョブをクラウドとローカルで同時本番運用しない
3. **チャンネル分離**:

   | チャンネル | 用途 |
   |---|---|
   | `#life-info` | P1-1 情報収集（RSS / YouTube） |
   | `#life-briefing` | P1-2 朝ブリーフ（予定 + タスク） |
   | `#life-reminder` | P1-3 リマインダー（食事・トレ・日課） |

4. **秘密管理** — トークン類は GitHub Secrets またはローカル env のみ。Vault には書かない。
5. **変更検証** — テンプレート・配信ロジックを変えたら `workflow_dispatch` でテスト配信して確認する。

---

## 3. アーキテクチャ

```
GitHub Actions（クラウド本番・統一済み）
│
├── p1-1-collect.yml       毎朝 06:32 JST  → #life-info
├── morning-briefing.yml   毎朝 06:00 JST  → #life-briefing
├── life-reminder.yml      各時刻（8種別） → #life-reminder

```

- Vault は Google Drive で同期 → GitHub リポジトリ（`naoyakousakabiz/ObsidianVault`）でバージョン管理

## 4. 通知テンプレート仕様

### 4.1 `#life-info`（P1-1 情報収集）

毎朝 06:32 に RSS・YouTube を収集し、1通で配信する。

| セクション | 内容 |
|---|---|
| 件数サマリ | Web / YouTube 各件数 |
| 🧠 AIニュース | 24h以内・最大5件・要約付き |
| 📰 Web記事 | 24h以内・最大10件・番号付き・要約+リンク |
| ▶️ YouTube | 24h以内・全件・番号付き・要約+リンク |

### 4.2 `#life-briefing`（P1-2 朝ブリーフ）

毎朝 06:00 に配信する。

```
⚡ Good Morning @naoya  —  YYYY年MM月DD日(曜)
──────────────────────
母より

1. 自分の人生を無駄にしない（命を大切に）
2. 死を恐れず、今日を全力で
3. 周りを大切に、伝えられるときに伝える
──────────────────────
📅  予定
• HH:MM〜HH:MM　予定名
──────────────────────
☑️  タスク
• [本業] タスク名
• [RINGBELL] タスク名
```

- **予定**: Google Calendar から gogcli 経由で取得（`GOG_TOKEN_B64` / `GOG_CREDENTIALS_B64`）
- **タスク**: 当日の Daily ノート（`80_Journal/01_Daily/YYYY-MM-DD.md`）から自動抽出
- **カレンダー失敗時**: 予定欄に `⚠️ 取得失敗 — GOG_TOKEN_B64 の期限切れの可能性あり` が表示される

### 4.3 `#life-reminder`（P1-3 リマインダー）

行動トリガに特化した短文通知。食事系は栄養ポイント、トレは週間メニューを同梱する。

| 時刻 (JST) | 種別 | 通知イメージ |
|---|---|---|
| 06:00 | training | 🔥 *Training — 今日のメニューを確認しよう* |
| 07:30 | breakfast | 🌅 *Breakfast — おはよう、しっかり食べよう* |
| 12:00 | lunch | 🍱 *Lunch — 午後に向けてチャージ* |
| 15:30 | snack | 🥜 *Snack — トレ・集中のためにチャージ* |
| 19:00 | work_end | 🚪 *Work End — 20時には切り上げてジムへ* |
| 20:00 | training（夜） | 🔥 *Training — 今日のメニューを確認しよう* |
| 22:00 | voice_journal | 🎙 *Voice Journal — 今日の声を残そう* |
| 22:30 | bedtime | 🌙 *Bedtime — そろそろ眠る準備を* |

### 4.4 夜の振り返り（Slack 通知なし）

**Slack への夜サマリは廃止**。振り返りは Obsidian 上で手運用する。

| 記録先 | 内容 |
|---|---|
| `80_Journal/01_Daily/YYYY-MM-DD.md` | 進捗・気づき・作業ログ |
| `80_Journal/02_Voice/YYYY-MM-DD.m4a` | 音声ジャーナル（Pixel Recorder → Drive） |

## 6. 日常のアクション

### 朝

1. `#life-briefing` で予定とタスクを確認し、今日の動きを確定する。
2. `#life-info` を2〜5分流し読みし、当日インプットの優先度を決める。
3. Cursor に「おはよう」で話しかけ、当日 Daily ノートを用意する。

### 日中

1. `#life-reminder` の通知に従って食事・補食・トレーニングを実行する。
2. 気づきや違和感は Daily ノートの「メモ・気づき」欄にその都度書く。

### 夜

1. Pixel Recorder で1〜2分、音声ジャーナルを録る。
2. Daily ノートの「メモ・気づき」を追記し、翌日タスクのたたき台を整える。

### 週次

1. 通知到達率・重複・誤配信を確認する（Actions タブ）。
2. 情報ソースを見直す（リンク切れ・フィード停止）。
3. `GOG_TOKEN_B64` の期限を確認する（予定欄に警告が出ていないか）。

## 8. 保留・バックログ

### 8.1 X 連携（保留：外部要因待ち）

**状態:** コード完成済み。有効化条件が整えば即日動く。

- **実装:** `scripts/p1-1-collect.py` の `fetch_x_entries()` に26アカウント分の収集ロジックが実装済み
- **有効化方法:** `X_BEARER_TOKEN` を GitHub Secrets に登録するだけ
- **壁:** X API Basic tier が必須（$100/月）。OSS スクレイパー・Bluesky代替は現状いずれも不安定または対象アカウントのカバー率が低く非現実的
- **再検討トリガー:** X が API 料金体系を変更した場合、または Bluesky への日本語 AI コミュニティの移行が進んだ場合

### 8.2 会話の自動データ化 PoC（バックログ：未着手）

**目的:** 対面での打ち合わせ・会話の録音を、手作業なしに要点として Vault に蓄積する。

**想定フロー:**
```
録音（Pixel Recorder など）
→ 文字起こし（Recorder 自動 or Whisper API）
→ 要点抽出（Claude API）
→ Daily ノートの「メモ・気づき」または専用ノートへ保存
```

**検討ポイント:**
- 録音方法: 対面ではノイズ・複数人声・同意の問題がある
- 文字起こし: Recorder の自動文字起こしが使えれば API コストなし
- 要点抽出: Claude API（1回数円）で十分対応可能
- プライバシー: 相手の同意・匿名化が前提

**再検討タイミング:** 音声ジャーナル（個人録音）の運用が定着してから。
