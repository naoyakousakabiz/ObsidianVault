# 08_Relationship コンテキスト

## 役割
人間関係の管理と対人パターンの学習。キャリア・人生の質に直結する人脈・関係性を構造化するフォルダ。

## フォルダ構成
| フォルダ | 内容 |
|---|---|
| `10_partner/` | パートナーとの価値観・合意事項・関係ログ |
| `20_family/` | 家族（両親・祖母・兄弟）との関係・近況・重要事項 |
| `30_network/` | 仕事関係の人脈（メンター・採用担当・元同僚・業界人） |
| `40_friends/` | 友人関係・コミュニティ |
| `50_patterns/` | 繰り返す対人パターン・学び・改善アクション |

## このフォルダでやること・やらないこと
- やること：関係性の記録、価値観の整理、人脈管理、対人パターンの学習
- やらないこと：同居ルール・インフラ契約（→`05_Life/40_agreements/`）、仕事の実務ナレッジ（→`01_Work/`）

## データ原則
- 個人情報は最小限に留め、「関係性・文脈・学び」を中心に記録する
- ネガティブな記録も「改善のための事実」として記録する（批判ではなくパターン抽出）
- 人脈ログは「最終接触日・文脈・次のアクション」をセットで管理する

## Claude Codeへの指示方針
- 人脈管理では「誰が・どんな価値を持つか・どう維持するか」の観点で整理すること
- 対人パターン分析時は過去の記録との一貫性を確認すること
- 個人情報の過度な詳細化は避け、洞察・行動方針に集中すること
