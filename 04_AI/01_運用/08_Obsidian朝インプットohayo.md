# 「おはよう」で朝の情報収集を全自動化する Claude Code スキル（ohayo）

毎朝30分かかっていた情報収集を、たった一言で完結させる設計メモ。

---

## 概要

Claude Code の「スキル」機能を使い、`~/.claude/skills/ohayo/SKILL.md` に自然言語でルールを記述するだけで実現できる朝のブリーフィング自動化。**コードを1行も書かずに**、複数データソースを並列取得して1つのレポートにまとめる想定。

---

## 何ができるか

`おはよう` と入力するだけで、以下を自動収集・要約する（接続済みのソースのみ）。

| カテゴリ | 取得元 | ポイント |
| --- | --- | --- |
| 今日の予定 | Google Calendar | 辞退済み・キャンセルを除外 |
| 未読メール | Gmail | 要対応メールだけ抽出、「即返信可能」にマーク |
| Slack未返信 | Slack MCP | メンション・DM・返信待ちのみ。Bot通知は除外 |
| 未完了タスク | Linear MCP | 期限切れ・本日期限を優先表示 |
| 健康データ | Oura Ring | 睡眠スコアが低ければ「今日は軽めに」と自動判断 |
| 昨日のセッション | ローカルJSONL | 何をしたか・未完了タスク・作成ファイルを自動分析 |
| MTGネクストアクション | tldv | 未実行のアクションアイテムを検出 |
| フォローアップ待ち | Gmail / Slack | 3日以上返信がない案件を自動検知 |
| MacBookの状態 | ローカル | バッテリー・ストレージ・CPU。異常時のみ表示 |
| AIニュース | Grok API | 今日の重要ニュース3件 |
| 天気 | Web Search | 傘が必要かどうかだけ |

最後に全データを統合して**「今日の最優先タスク3つ」**を自動判定する。

## アーキテクチャ設計

### ディレクトリ構成

```
~/.claude/
└── skills/
    └── ohayo/
        └── SKILL.md   ← ここに自然言語でルールを書くだけ
```

### SKILL.md の構造（例）

```markdown
---
name: ohayo
description: 朝のブリーフィングを自動生成するスキル
user-invocable: true
---

## トリガー
「おはよう」「おはようございます」「morning」

## 取得するデータソース
1. Google Calendar（gog MCP）
2. Gmail（gog MCP）
3. Slack未返信（Slack MCP）
…

## 出力フォーマット
### ☀️ [日付] 朝のブリーフィング
…

## フォールバック
- 各ソースが失敗した場合はスキップ
- 全部失敗しても天気と日付は必ず出力
```

### 並列実行のイメージ

Claude Code の **Agent tool** を使い、複数データ取得を同時並列で実行する。

- **Sequential（従来）:** 各ソース30秒 × 11 ≒ 約5分
- **Parallel（このスキル）:** 全ソース同時 ≒ 数十秒台を狙う

```
Agent 1 ─── Google Calendar ──┐
Agent 2 ─── Gmail ────────────┤
Agent 3 ─── 天気検索 ─────────┤
Agent 4 ─── Grok API ─────────┼──→ 統合 → ブリーフィング出力
Agent 5 ─── セッションログ ───┤
Agent 6 ─── Linear ───────────┘
```

## セッションログ分析（差別化ポイント）

Claude Code はセッションの会話を **JSONL形式** で `~/.claude/projects/` に保存している。このスキルはそのファイルを解析する想定。

### 抽出する情報

- `type: "user"` → 何を指示したか（作業内容の把握）
- `type: "assistant"` → `tool_use` の種類と回数（作業量の把握）
- タイムスタンプ → 開始・終了時刻（所要時間の計算）
- `file_path` 引数 → 作成・編集したファイルの一覧

### 出力例

```
昨日のセッション（3件）
├─ claude-env-dashboard  完了 ✓  GitHub push済み
├─ 契約書3点             完了 ✓  Slack送信済み
└─ Claude Code活用事例集 完了 ✓  外部配布版に一般化完了

未完了タスク: なし
```

**注意:** セッションログのJSONL形式は公式ドキュメント外の仕様。Claude Code のアップデートで変わる可能性がある。

## フォールバック設計

「1つ壊れても全体が壊れない」設計。

| 状況 | 挙動 |
| --- | --- |
| gog 認証切れ | メール・カレンダーをスキップ |
| Oura 未連携 | 健康データをスキップ |
| Linear 未接続 | タスクをスキップ |
| 全ソース失敗 | 天気と日付は必ず出力 |

---

## 設計の背景：なぜこの項目か

過去1週間のセッションログを分析した結果、朝一番の指示で多かったのは次の2パターン。

1. **「昨日の〇〇のログを調べて」** → 昨日の文脈引き継ぎ
2. **「tldv見てネクストアクション確認して」** → MTGの言いっぱなし検知

朝の本質は「**昨日の続きを思い出す**」と「**やり忘れを検知する**」の2つ。メールやカレンダーはスマホでも見られるが、**Claude Code セッションで何を決めたか** と **MTGで言った「やります」が実行されたか** はこのスキルで取りにいく価値が大きい。

## セットアップ手順（5分）

### 1. フォルダ作成

```bash
mkdir -p ~/.claude/skills/ohayo
```

### 2. SKILL.md を作成

`~/.claude/skills/ohayo/SKILL.md` に以下の要素を記述する。

- トリガーワード: `おはよう` / `おはようございます` / `morning`
- データソース（接続済みのものだけ使う）: Calendar / Gmail / 天気 / セッションログ 等
- 出力フォーマット: `[天気] → [今日の予定] → [メール] → [昨日のログ] → [最優先タスク3つ]`
- フォールバック: 各ソース失敗時はスキップ。天気と日付は必ず出力

### 3. MCP を接続（任意）

| MCP | 取得できる情報 |
| --- | --- |
| gog（Google連携） | Calendar / Gmail |
| Slack MCP | 未返信メッセージ |
| Linear MCP | 未完了タスク |

MCP 未設定のソースはスキップされる想定。**最初はカレンダーとメールだけでも十分**という運用も可。

### 4. 実行

Claude Code を開いて「おはよう」と入力する。

## 限界と正直な話

- **毎朝使えているか:** 現状は毎朝ではない。PC を開いてから Claude Code を起動するまでの習慣がまだない。
- **完全自動実行は難しい:** `launchd` で `claude -p` を叩く方法は、OAuth 認証がバックグラウンドで通らず失敗しやすい、という知見。
- **セッションログ形式は非公式:** アップデートで動作が変わる可能性あり。

現状の最善策は **「PC を開いたら最初に『おはよう』と打つ」** という習慣化。音声入力（Typeless + Shokz 等）と組み合わせると使いやすい。

## まとめ

| Before | After |
| --- | --- |
| メール・Slack・カレンダーを1つずつ確認（約30分） | `おはよう` と打つだけ（約30秒） |
| 昨日の作業を記憶から思い出す | セッションログから自動サマリー |
| MTGの「やります」を忘れる | tldv 連携で未処理アクションを検知 |
| 返信催促のタイミングを見逃す | 3日以上返信なし案件を検出 |

> 情報を集めること自体に価値はない。集めた情報から何をするか決めることに価値がある。  
> このスキルは前者を自動化して、後者に集中する時間を作る。
