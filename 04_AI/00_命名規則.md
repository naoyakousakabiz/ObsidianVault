# 04_AI 命名規則（共通）

## 目的
- `04_AI` 配下の命名を統一し、検索・移管・昇格を簡単にする。
- 「似た名前が多くて迷う」状態を防ぐ。
- 人が説明した内容から、AIが格納先とファイル名を一意に判定できる状態にする。

## まずこれだけ（最短ルール）
1. その内容は「長期の正本」か、「再現手順」か、「未昇格メモ」かを決める。
2. 下の判定表でフォルダを1つに決める。
3. `NN_内容.md` の型で命名する（`NN` は同階層の空き番号）。
4. 迷ったら `03_学習` に置き、週次で昇格判定する。

## 格納先の一発判定（AI向け）
| 条件（Yesなら） | 格納先 |
|:--|:--|
| Vault全体の方針・運用ルール・ガバナンスか | `01_運用/` |
| 他案件でも再利用できる再現手順か | `02_手法/` |
| 講座メモ・要約・未整理の学習素材か | `03_学習/` |
| ドメイン非依存の雛形・テンプレか | `04_テンプレート/` |
| SNS発信（X / YouTube / note）のノウハウか | `05_SNSノウハウ/` |
| 参照のみで更新しない旧資料か | `99_アーカイブ/` |

## 入力テンプレ（説明を渡すとき）
以下の5項目を1-2行で渡せば、格納先と命名が決まる。

- 目的: 何のための内容か
- 再現性: 他案件で再利用できるか（Yes/No）
- 状態: 正本 / 未昇格 / アーカイブ
- 対象: AI全般 / ClaudeCode / NotebookLM / 連携設計 / SNS
- 種別: ルール / 手順 / メモ / テンプレ

## 命名規則（AI判定しやすい最小形）
- 基本形: `NN_内容.md`
- ルール文書: `00_運用ルール.md`
- 入口文書: `00_索引.md`
- 実行ハブ: `01_実行計画.md`
- 基礎設定: `02_基礎設定.md`

### `02_手法` のみの補助規則
- 再現手順ファイルは、内容が伝わる複合名を許可する。
- 形式: `NN_カテゴリ_主題.md`（例: `21_ClaudeCode実装_Remotion動画テロップ自動化.md`）
- カテゴリは `AI全般 / ClaudeCode / NotebookLM / 連携設計` のみ使用。

## フォルダ命名（最小）
- 先頭は2桁番号 + `_` + 用途名
- 現在の固定レイヤー
  - `01_運用`
  - `02_手法`
  - `03_学習`
  - `04_テンプレート`
  - `05_SNSノウハウ`
  - `99_アーカイブ`

## ファイル命名（共通）
- 基本形: `NN_内容.md`
- 「ルール文書」は各フォルダで `00_運用ルール.md` に統一
- 入口ファイルは `00_索引.md` を優先

## 学習フォルダ（`03_学習`）の最小構成
- `00_運用ルール.md`
- `01_実行計画.md`
- `02_基礎設定.md`
- `10_リサーチ/`
- `20_講座・セミナー資料/`

## 講座・セミナー資料
- `20_講座・セミナー資料/` 配下に集約
- 同一出典で3件以上なら `出典名_期間/` のサブフォルダ化
- ファイル名は主題優先（出典・日付は本文冒頭に明記）

## 手法フォルダ（`02_手法`）
- 再現可能手順のみを格納
- サブ分類
  - `10_AI全般/`
  - `20_ClaudeCode/`
  - `30_NotebookLM/`
  - `40_連携設計/`

## 禁止事項
- `Untitled` のまま保存しない
- 同一階層で同じ番号をファイルとフォルダに重複使用しない
- 同じ内容を別名で複製しない（1テーマ1正本）
