# 運用ルール

**日次・週次・月次のAI活用ルーティン、チェックリスト、参照リンクをこの1ファイルに集約する。**  
（ファイル名：`10_運用ルール.md`｜旧：`10_AI運用ルール.md`／`10_日次週次AI運用ルール.md`／`10_日次週次ルール.md`）

> ツール役割の詳細は `01_ツール役割と運用.md`、Phase・品質ゲートは `02_フェーズとガバナンス.md` を正とする。

## 新規ノートを増やすとき（ミニルール）

**正本以外**にノートを新設・追記するときは、**詳細を複数ファイルに書き分けない**。

- **基本形:** 概要 **1〜3 行** ＋ **該当する正本（または索引）へのリンク**
- **本文を厚く書くのは**、そのトピックの**正本として決めた1ファイルだけ**（`04_AI/README.md` の記載責任マトリクス・各ドメイン `CLAUDE.md` に従う）
- 迷ったら `00_Inbox/` や `80_実験ログ/` に置き、週次で昇格先を決める

## 週次ルーティン（毎週末）

### 1. フェーズ・KPI（週次）

**必須（◎）** ― 週末までに必ず（「100点運用」の週末最低ラインとも整合）

1. `02_フェーズとガバナンス.md §20` にKPIを記録する  
2. **音声ジャーナル（Voice）のリネーム：** Google Drive 同期が取れたあと、`80_Journal/02_Voice/` 内の **ISO 形式以外の `.m4a`** を **`YYYY-MM-DD.m4a`** に揃える（Vault ルートで `python3 99_System/scripts/rename_voice_m4a.py` または手動）。**手順・NL の役割**は Vault 根 **`CLAUDE.md`（音声ジャーナル）**／**`80_Journal/CLAUDE.md`** を参照。  
3. `00_Inbox/` に未整理があればフォルダへ仕分ける（**ゼロ件ならスキップ可**）

**任意（◇）** ― 余力・必要性に応じて

4. 今週のAI活用の気づきを `04_AI/80_実験ログ/` に1行でも残す  
5. NotebookLMソースの棚卸し（古い資料は除外）

### 2. 5資本 週次振り返り（80_Journal に記録）

| 資本 | 確認観点 |
|---|---|
| ①生活基盤資本 | 睡眠・運動・食事・ストレス管理が崩れていないか |
| ②関係基盤資本 | 家族・友人・コミュニティに接点があるか |
| ③経済安定資本 | お金の仕組みが回っており、不安に思考を奪われていないか |
| ④身体・印象資本 | 清潔感・ボディメイク・美容に継続的に取り組めているか |
| ⑤成長・充足資本 | 目標意識・学習・新たな挑戦で前進感を得られているか |

## 月次ルーティン（月初〜月中で1回）

**目的:** エージェント・人間ともが「正本と入口」を信頼できる状態を維持する（索引と実ファイル・パス正字のズレは検索コストと誤出力に直結する）。

1. **パス・表記の正字（Vault 根）**  
   - `CLAUDE.md` の **「パス・表記の正字」** を開き、現行フォルダ名と矛盾がないか確認する（特に **`02_Business/`** と **`03_Career/`** の取り違えがないか）。

2. **索引 ↔ 実ファイルの照合**  
   **対象パスのマスター一覧**は Vault 根 `CLAUDE.md` の **「索引マスター（月次照合対象）」** を正とする。各索引に載っているファイル名が Vault 上に実在するか目視で確認する。リネーム・追加・削除した月は必ず索引側を更新する。

3. **正本リストの追随**  
   - `04_AI/README.md` の「正本ファイル」表が、Vault 根の索引マスター・各正本の所在と大きく食い違っていないかざっと見る（パス付き一覧の正本は README のみ。`04_AI/CLAUDE.md` は誘導・入口のみ）。  
   - **新規 `00_索引`・索引のリネーム**の追記・同セッション完了は、下方「100点運用」**§2** に従う（ここでは繰り返さない）。

4. **記録（任意）**  
   - 修正した項目があれば、`04_AI/80_実験ログ/` に1行でもよいので残す（いつ直したかの根拠になる）。

## 設計を「100点運用」に仕上げるルール（構造は増やさない）

採点で残りがちなギャップは、**フォルダや正本の数**ではなく、**毎週・毎月の実行の抜け**である。複雑化させず、次を**完了定義**とする。

### 1. 月次照合の固定化（カレンダー1本）

- **いつやるか**を1つ決め、**カレンダーの繰り返し予定**に登録する（例：毎月第1月曜の朝5分／暦月1日　など。**自分の負荷が最小のスロットでよい**）。
- 実行内容は **「月次チェック（コピペ用）」のチェックボックスだけ**。項目を増やさない。

### 2. 索引・リネームの「同セッション完了」

- **`00_索引.md` の一覧変更**や、索引に載っている**ファイルのリネーム・削除**をした場合、**同じ作業セッション内で** Vault 根 `CLAUDE.md` の **「索引マスター（月次照合対象）」** にパスが揃っているか確認する（新規索引なら**その場で1行追加**）。**「あとで」と残さない。**

### 3. 週末が重い週の「最低限3つ」

時間がない週は、週末品質チェックのうち次の**3つだけ**実行すればよい（他は翌週へ）。順不同。

1. KPI週次（`02_フェーズとガバナンス.md §20`）  
2. **`80_Journal/02_Voice/`** のリネーム、または「今週対象なし」と確認  
3. **翌日へのデイリー繰り越し**が破綻していないかざっと見る（デイリーノートの「今日やること」）

### このVaultにおける「100点」の定義

- 上記 **1〜3** を遵守し、**索引マスター・各 `00_索引`・実ファイルにズレがない状態が続いている**こと。  
- 数値目標は**3ヶ月連続で月次をスキップしない**程度で十分（Perfect more は不要）。
