# データ蓄積ガイド

各フォルダに蓄積すべきデータの一覧。「第二の脳」と「業務自動化」の両立を目的とする。

**原則：生ログではなく、再利用できる形に整えてから保存する。**

### 診断・検査・適性の網羅（インデックス運用）

「受けた／受けうる診断・検査・評価」を **カテゴリで棚卸し** し、漏れ・再検忘れ・AIへの文脈抜けを防ぐ。**未実施・該当なしは空欄でよい**（**可能性リスト**として残す）。**原本（PDF・紙）は別管理**でもよいが、Vault 側には **①実施日 ②機関（ざっくり）③結論（型名・数値は要約）④次検査・要精密・紹介状**。**数値の羅列・識別子**は伏せ字・レンジ（`02_フェーズとガバナンス`）。

#### 可能性リスト（細目）— 該当なしでも読んでチェック

- **心血管：** 血圧・脂質・血糖の**ホームログ**、家族性高コレステロールの検討  
- **腎・尿：** eGFR 推移、尿路結石、蛋白尿  
- **消化：** ピロリ、肝線維化スコア、**NAFLD／脂肪肝**の画像要約  
- **眼：** ドライアイ評価、コンタクト処方、老視進行  
- **耳鼻：** めまい・前庭機能（**要約のみ**）、難聴フォロー  
- **皮膚：** アトピー、接触皮膚炎、光線角化症の経過  
- **生殖・性：** 男性ホルモン、精液検査（該当時）、**妊活関連（該当時・機微）**  
- **腫瘍：** がん検診の**種類／結果区分**（要精密・フォロー期限）  
- **労働・法令：** **長時間労働者面接**、心の健康支援（事業主実施の面談メモ要約）  
- **技能：** 運転免許の**条件付き・更新時健康診断の期限メモ**（結果の細部は不要なら省略可）  
- **低再現性の「診断」：** アプリの性格診断・タイピング診断等は **エンタメ枠** と明記し、**キャリア判断の主証拠にしない**

> **運用：** `11_health` に **診断・検査インデックス1枚** を置き、**A〜L ＋ 家族・環境** を **チェックボックス** で**年1更新**。新規受診のたび **1行追加**。NotebookLM は **要約のみ**。**精神・職域・家族・生殖は特に機微** — ノート先頭に **利用目的（自分／面談／AI）** を書く。

## エージェント拡張性の高いデータ（プロのAI活用家向け）

**目的：** 同じ説明を毎回しなくてよくなり、**誤前提・ツール取り違え・ガバナンス事故**を減らす。「趣味のメモ」ではなく **推論コストを下げる仕様書** に近い情報を置く。

**設計原理（3つだけ）：**

1. **一度書けば百回使う** — エージェントが毎回聞きそうな「既定値」を外部化する。  
2. **禁止と既定を対で書く** — 「やってほしいこと」より **「やってはいけないこと」＋「デフォルト」** のほうが事故が減る。  
3. **出典パスを1行で指せる** — 「あのファイルの3節」ではなく **ノート1枚＝1責務** でリンク可能にする。

| データ種別 | 具体例・中身 | 置き場の目安 |
|---|---|---|
| **コンテキスト辞書（コードネーム）** | パートナー・本業・クライアント層を **P／A／C 等の略号** にし、「このVaultでは何を指すか」1表に固定。**第三者への出力禁止名** も併記 | `99_System` または `04_AI` 直下の短い1枚（**秘密は書かない**） |
| **出力仕様プロフィール** | 返答の長さ、表・箇条書きの偏好、根拠の出し方（推定か引用か）、日本語のトーン、コードを出すときの言語 | `04_AI` または Cursor ルールと **相互リンク** |
| **判断アーキタイプ** | リスク許容、費用対効果の閾値、「決め切り」vs「保留がデフォルト」、後悔しやすいパターン | `80_Journal` の意思決定ログとリンク |
| **時間リズム・不可侵枠** | 週の骨格（固定予定）、反答が遅い時間帯、締切の多い曜日。スケジュール提案AIの前提になる | `05_Life/20_events` またはデイリー運用ノートとリンク |
| **並行ストリーム索引（1行×1プロジェクト）** | 「いま動いている案件」の **名称・目的・次アクション・Vaultパス**。文脈切り替えのオーバーヘッドを削る | `00_Inbox` の週頭サマリ or 各ドメイン `00_index` |
| **データ分類の短文マップ** | レベルA/B/C（機微度）の例と、「このツールに貼ってよい／NG」。**`02_フェーズとガバナンス` の正本を1行で指す** | `04_AI` のガバナンス節へリンクする1枚 |
| **失敗カタログ（ネガティブ few-shot）** | AIが **誤解した前提** と、そのときの **修正プロンプト／ルール追記**。再発防止が効く | `04_AI` 実装ログ or `80_実験ログ` |
| **連携インベントリ（非秘密）** | MCP・OAuth・定期ジョブが触る **サービス名一覧**（トークン・パスワードは書かない）。「何が自動化可能か」の地図 | `04_AI` MCPメモと統合してよい |
| **再利用テンプレの正本パス** | 提案文・DM・議事メモ・週報の **最新版ファイル** だけを列挙（バージョン混乱を防ぐ） | 各ドメイン `Templates` または `00_index` |
| **用語・ドメイン辞書** | 業界略語、社内用語、RINGBELL 固有語、**プロダクト名の表記ゆれ統一** | `01_Work` / `02_Business` に分けてもよい |
| **自分史（30秒版）** | 転機・信条・いまの焦点を **1段落**。面接・自己紹介・ブログの初稿生成の固定ソース | `03_Career/10_プロフィール` 付近 |
| **依頼テンプレ（Intent テンプレ）** | 「よく頼む作業」の **入力スロット定義**（例：目的・制約・出力量・参照ファイル）。Cursor/Claude のコピペ運用が速くなる | `04_AI` または `.claude/commands` の説明とリンク |

> **100点を狙うなら：** 上表はすべて **「エージェントが検索1発で拾える粒度」** にし、長文は **章立てした子ノートに分割** する。README と索引（`00_index`）から **2ホップ以内** で辿れる状態を維持する。
