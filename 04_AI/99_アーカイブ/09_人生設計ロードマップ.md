## 0. 診断：なぜ知識は腐るのか

多くの知識管理システムが崩壊する理由は設計の質ではない。**処理に強制力がないから**。

```
よくある失敗パターン
  インプット増加 → Inbox溜まる → 処理が追いつかない
       → 「いつかやる」 → 腐る → システム全体への不信
```

解決策は1つ：**処理を「選択」から「仕組み」に変える。**

## 3. Readwise 導入ガイド

### なぜReadwiseか
Kindleプラグインより**はるかに強力**。Kindle・Web記事・Twitter/X・PDFのハイライトを一元管理し、Obsidianへ自動同期する。月$7.99（年払い$4.99/月）。

### セットアップ手順

**Step1：Readwise契約**
- readwise.io でアカウント作成（30日間無料トライアルあり）
- プラン：Reader（$7.99/月）またはLite（$4.99/月・Reader機能なし）
- **推奨：Reader付き**（Web記事のハイライト管理が格段に便利になる）

**Step2：Kindle連携**
- Readwise → Settings → Kindle → Amazonアカウントと連携
- 既存のKindleハイライトが自動インポートされる

**Step3：Obsidian連携**
1. Obsidianの設定 → コミュニティプラグイン → 「Readwise Official」を検索・インストール
2. プラグイン設定でReadwise APIキーを入力（readwise.io/access-tokenで取得）
3. 同期先フォルダを `20_Input/01_Kindle/` に設定
4. 同期スケジュール：「自動（Obsidian起動時）」に設定

**Step4：Web記事のハイライト（Readwise Reader）**
- ブラウザ拡張「Readwise Reader」をインストール
- Web記事を読みながらハイライト → 自動でObsidianに同期
- 従来の`20_Input/02_Clippings/`への手動クリッピング作業が不要になる

**Step5：既存のKindleプラグインの扱い**
- Readwise導入後はKindleプラグイン（obsidian-kindle-plugin）は無効化してOK
- Readwiseの方が同期精度・UI・検索性が上

### Readwise後の情報フロー

```
Kindle読書 → Readwise → 20_Input/01_Kindle/ → Claude Code処理 → 各ドメインへ
Web記事    → Readwise Reader → 20_Input/01_Kindle/ → Claude Code処理 → 各ドメインへ
```

## 4. インプットを腐らせない仕組み

### 核心：週次処理儀式（毎週固定）

```
毎週日曜 20:00〜20:30（30分・絶対固定）

Step1（10分）: 00_Inbox を処理
  → Claude Codeに「Inboxを処理して適切なフォルダに振り分けて」

Step2（10分）: 20_Input を処理
  → Claude Codeに「Kindleとクリッピングを処理して各ドメインへ転記して」

Step3（10分）: 週次振り返りを作成
  → Claude Codeに「今週のログから週次振り返りを80_Journalに作って」
```

**このリチュアルを守るだけで、知識は腐らない。** 週30分だけ。

### インプット時の「Output First」ルール

何かをキャプチャする前に一言決める：
```
「このインプットは→何のために→どこに保存するか？」
```

| インプット | アウトプット先 | 例 |
|---|---|---|
| 転職関連の記事 | `03_Career/30_キャリア戦略/` | 面接対策に使う |
| RINGBELL業務改善のアイデア | `02_Business/RINGBELL/` | CS対応に使う |
| AI活用の新手法 | `04_AI/03_手法/` | Claude Code自動化に応用 |
| 投資・財務の知識 | `05_Life/30_finance/` | 月次見直しに使う |
| 行き先が決まらない | **保存しない** | — |

## 10. 日常運用の最小ルール

**これだけ守れば、3年後のVaultは別次元になる。**

```
毎朝3分  : 前日の気づきを00_Inboxに音声入力（Typeless）
毎晩2分  : その日の重要な出来事を1行メモ
毎週30分 : 週次処理儀式（日曜20時・固定）
毎月1時間 : NotebookLMで月次パターン分析 → 80_Journalに保存
```
