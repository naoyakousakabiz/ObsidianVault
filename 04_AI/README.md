# 04_AI 運用ガイド

このフォルダは「AI戦略の正本」「業務自動化」「手法ナレッジ」「一時メモ」「退避資料」を分けて管理する。

**索引・正本ルール・どれをいつ読むか**は、このREADMEに集約する。

---

## フォルダ構成

- `01_ツール運用`：戦略・運用の正本（長期運用）
- `03_手法`：読書術/ファクトチェックなど手法ナレッジ
- `04_学習`：外部講座・セミナー資料（ClaudeCode講座等）
- `80_実験ログ`：一時的な実行計画・短期メモ
- `99_archive`：旧版・統合版・Deprecated資料

※ 業務自動化設計はドメインフォルダ配下に配置（`01_Work/50_AI自動化/`・`02_Business/RINGBELL/50_AI自動化/`・`03_Career/50_自動化/`）

---

## どこに書くか（判断基準）

- 方針/ルール/全体設計を書きたい -> `01_ツール運用`
- AI自動化の「やりたいこと・アイデア」を貯めたい -> `01_ツール運用/06_AI自動化_やりたいことバックログ.md`
- 業務フローや自動化設計を書きたい -> 各ドメインの `50_AI自動化/`（`01_Work/50_AI自動化/` 等）
- スキル・手法・型を蓄積したい -> `03_手法`
- 外部講座・セミナーの参照資料 -> `04_学習`
- 明日や今週だけ使う計画 -> `80_実験ログ`
- もう更新しないが参照は残したい -> `99_archive`

---

## 整合性ルール（正本定義）

1. ツール名・役割定義は `01_AI活用整理.md` を正とする
2. Phase進行・完了定義は `02_AIキャッチアップ.md` を正とする
3. データガバナンス・品質ゲートは `02_AIキャッチアップ.md` を正とする
4. 入力系AIは `Claude` を標準とし、`Gemini` はGoogleサービス連携時、またはNotebookLMソース前提の補助推論時に限定する
5. 実績数値・対外向けトークは **`03_Career` 等**で管理する（定型枠の参考は `99_archive/AI戦略_02_実績と対外説明.md`）
