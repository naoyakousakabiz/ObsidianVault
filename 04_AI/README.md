# 04_AI 運用ガイド

このフォルダは「AI戦略の正本」「業務自動化」「手法ナレッジ」「一時メモ」「退避資料」を分けて管理する。

**索引・正本ルール・どれをいつ読むか**は、このREADMEに集約する。

---

## フォルダ構成

- `01_ツール運用`：戦略・運用の正本（長期運用）
- `02_AI導入コンサルテンプレートフォルダ`：**ドメイン非依存**の AI導入・コンサル用テンプレ正本（ルート番号: **`00`** メタ・**`01`** 司令塔・**`02`** マスター一覧・**`03`** 納品チェック）。各ドメインの `50_AI自動化/` はここをコピー起点にできる
- `03_手法`：読書術/ファクトチェックなど手法ナレッジ
- `04_学習`：外部講座・セミナー資料＋**実行ハブ `今後やること.md`**（30日・Phase・継続）
- `80_実験ログ`：一時的な実行計画・短期メモ
- `99_archive`：旧版・統合版・Deprecated資料

※ **ドメイン固有**の業務フロー・自動化の実体・数値は各フォルダ配下（例: `02_Business/RINGBELL/50_AI自動化/`、`01_Work/50_AI自動化/`）

---

## どこに書くか（判断基準）

- 方針/ルール/全体設計を書きたい -> `01_ツール運用`
- AI自動化の「やりたいこと・アイデア」を貯めたい -> `01_ツール運用/06_自動化バックログ.md`
- AI導入コンサルの**汎用テンプレ**（型・ロードマップ・チェックリスト）を変えたい -> `02_AI導入コンサルテンプレートフォルダ/`
- 業務フローや**ドメイン固有**の自動化設計・施策実体を書きたい -> 各ドメインの `50_AI自動化/` 等
- スキル・手法・型を蓄積したい -> `03_手法`
- 外部講座・セミナーの参照資料 -> `04_学習`
- 講座＋Phase＋**毎週の蓄積・ルーティン**まで1枚で追う -> `04_学習/今後やること.md`
- 明日や今週だけ使う計画 -> `80_実験ログ`
- もう更新しないが参照は残したい -> `99_archive`

- ツール運用フォルダの全体像・読み順：`01_ツール運用/00_索引.md`
