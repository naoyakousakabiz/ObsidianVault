# Claude Code スキル作成 指示文集（28種）

> **使い方：** 各指示文をそのまま Claude Code に貼り付けて実行する。
> スキルファイルは `.claude/skills/` 配下に `スキル名.md` として保存される。

---

## 前提：スキルファイルの構造について

各スキルは以下のフォーマットで `.claude/skills/` に保存する。

```
.claude/skills/
├── post-meeting.md
├── generate-proposal-excel.md
├── ...
```

スキルファイルの基本構造：
```markdown
# スキル名

## 概要

## 前提条件
必要なツール・環境・認証情報

## 手順
Step-by-step の処理内容

## 出力
成果物の仕様
```

## スキルの目的
会議の文字起こしテキストを受け取り、5つの成果物を全自動で生成・配信する「会議後自動化」スキル。

## 処理フロー（この順番で実行すること）
1. 入力受付
   - /lp（クリップボード）または /lf（ファイル指定）で文字起こしテキストを受け取る
   
2. クライアント自動判定
   - テキスト内の固有名詞・会社名・プロジェクト名を抽出
   - CLAUDE.md の顧客リストと照合してクライアント名を特定
   - 不明な場合はユーザーに確認を求める

3. カレンダー補完
   - Google Calendar MCP で該当会議の情報（日時・参加者・件名）を取得
   - 文字起こしに欠落している情報を補完する

4. 議事録生成（2形式）
   - プレーンテキスト版（.md）：要点・決定事項・TODO・ネクストアクションを構造化
   - フォーマット済み版（.docx）：会社テンプレートに準拠したWord形式
   - 保存先：`clients/{顧客名}/minutes/{YYYY-MM-DD}_{会議名}.md` および同 `.docx`

5. Gmail下書き作成
   - gogcli（Gmail）で参加者への御礼メールを下書き作成
   - 件名：「【御礼】{会議名} - {日付}」
   - 本文：参加御礼・議事録添付案内・ネクストアクション記載

6. Slack社内報告
   - Slack MCP で #meeting-notes チャンネルに投稿
   - 投稿内容：会議概要・決定事項・TODO（担当者・期限付き）

7. Notion DB更新
   - Notion MCP で Meetings DB に新規レコード追加
   - フィールド：日付・クライアント名・会議名・参加者・決定事項・TODO・議事録リンク

## 成果物一覧
| # | 成果物 | 形式 | 保存先 |
|---|--------|------|--------|
| 1 | 議事録（プレーンテキスト） | .md | clients/顧客名/minutes/ |
| 2 | 議事録（フォーマット済み） | .docx | clients/顧客名/minutes/ |
| 3 | 御礼メール下書き | Gmail下書き | Gmailの下書きフォルダ |
| 4 | Slack社内報告 | Slackメッセージ | #meeting-notes |
| 5 | Notion DB更新 | DBレコード | Notion Meetings DB |

## 依存ツール
- gogcli（Gmail操作）
- Google Calendar MCP
- Notion MCP
- Slack MCP
- python-docx（.docx生成）

## エラーハンドリング
- 各ツールが未接続の場合は該当ステップをスキップし、ユーザーに通知する
- クライアント判定が不確かな場合は必ず確認を取ってから続行する
- 全成果物の生成完了後にサマリーを表示する

## スキルの目的
クライアントへのヒアリング内容（テキスト or 音声文字起こし）から、
コンサルティングレベルの AI 活用提案書 Excel（11シート構成）を一発自動生成する。

## 入力
- ヒアリング内容のテキスト（/lp または /lf で受け取る）
- クライアント名・業種・規模（不明な場合はユーザーに確認）

## 出力：11シート構成 Excel ファイル

### Sheet 1：サマリー
- タイトル・クライアント名・作成日・作成者
- 事業概要（ヒアリングから自動生成）
- カテゴリ別課題と概算費用の一覧表

### Sheet 2：AI活用課題一覧
- ヒアリングから抽出した課題を行単位でリスト化
- 各行の列：課題名 / カテゴリ / 優先度（高中低）/ 難易度（1-5）/ AI解決策 / 期待効果 / 工数（人日）/ 概算費用
- チェックボックス列を先頭に配置

### Sheet 3：お見積り（統合）
- 総額・総工数・単価を見出しに表示
- カテゴリ別セクションに分けた費用内訳
- 自動集計式（SUM）を全合計セルに設定

### Sheet 4〜8：見積り詳細 A〜E（カテゴリ別）
- カテゴリ名（Sheet2の分類に対応）
- 各施策ごとに：現状課題 / AI解決策 / 期待効果 / 工程別工数表
- 工程：要件定義・設計・開発・テスト・運用 の5列

### Sheet 9：実行ロードマップ
- 24週間のガントチャート
- 施策ごとに開始週・終了週を色付きセルで表現
- マイルストーン行を別色で強調

### Sheet 10：プロジェクト体制
- 支援側（自社）の役割と担当者名
- クライアント側の役割と担当者名
- テーブル形式で体制図を表現

### Sheet 11：次回会議アジェンダ
- 会議情報（日時・場所・参加者）
- アジェンダ表（番号・議題・担当・時間・資料）
- 今回の目標と準備依頼事項

## 実装方針
- Python の openpyxl ライブラリを使用して Excel を生成
- セルの書式設定（太字・背景色・罫線・数式）を適切に実装
- ファイル名：`{クライアント名}_AI提案書_{YYYY-MM-DD}.xlsx`
- 保存先：`clients/{クライアント名}/proposals/`

## エラーハンドリング
- ヒアリング内容が不足している場合は補完すべき項目を列挙してユーザーに確認
- openpyxl が未インストールの場合は `pip install openpyxl` を自動実行

## スキルの目的
どんな種類の会議にも対応できる汎用議事録生成スキル。
2段階ワークフローで、入力の種類・会議の性質を問わず7つの成果物を生成する。

## 2段階ワークフロー

### Stage 1：入力解析・構造化
1. 入力を受け付ける（以下の複数形式に対応）
   - テキスト貼り付け（/lp）
   - テキストファイル（/lf）
   - .vtt / .srt 字幕ファイル（/lf）
2. 会議の種類を自動判定する
   - 社内会議 / クライアント会議 / 採用面接 / 1on1 / キックオフ / 定例 / ブレスト
3. 参加者を抽出・役割を推定する
4. タイムライン（発言の流れ）を構造化する

### Stage 2：7成果物の生成
1. **要約版議事録**（.md）：決定事項・TODO・ネクストアクションのみ抽出
2. **詳細版議事録**（.md）：発言ベースの完全な議事録
3. **Word版議事録**（.docx）：フォーマット済みの提出用
4. **TODO一覧**（.md）：担当者・期限付きのタスクリスト
5. **御礼/共有メール下書き**：会議種別に応じたテンプレートで生成
6. **Slack投稿文**：社内共有用の簡潔なサマリー
7. **カレンダー登録用テキスト**：ネクストアクションの日程情報

## 会議種別ごとの出力カスタマイズ
- 社内会議：TODO重視、Slack投稿必須
- クライアント会議：御礼メール重視、フォーマル文体
- 採用面接：評価コメント欄を追加、人事用フォーマット
- 1on1：成長メモ・フィードバック欄を追加
- ブレスト：アイデア分類（採用・保留・廃案）を追加

## 保存先
- `minutes/{YYYY-MM-DD}_{会議名}/` 配下に全成果物を格納

## スキルの目的
Zoom・録画・音声ファイルを投げるだけで議事録を自動生成する。
対応ファイル形式：.txt（文字起こし済み）/ .mp4 / .mov / .vtt

## 処理フロー

### Step 1：ファイル受付・形式判定
- /lf でファイルパスを受け取る
- 拡張子を判定して処理を分岐

### Step 2：文字起こし処理（形式別）
- **.txt**：そのまま次のステップへ
- **.vtt / .srt**：タイムスタンプ・話者情報を保持しながらテキストに変換
- **.mp4 / .mov**：
  1. ffmpeg で音声を抽出（.wav に変換）
  2. Whisper（openai-whisper）で文字起こし実行
  3. 話者分離が可能な場合は pyannote.audio を使用

### Step 3：議事録生成
- 文字起こしテキストを構造化分析
- 以下を自動抽出：
  - 会議目的・アジェンダ
  - 参加者と発言割合
  - 決定事項（明示的な合意・承認）
  - TODO（担当者・期限）
  - 懸念事項・リスク
  - ネクストアクション

### Step 4：成果物出力
1. 議事録 .md（要約版）
2. 議事録 .docx（提出用フォーマット）
3. 全文テキスト .txt（文字起こし原文）
4. TODO一覧 .md

## 依存ライブラリ
- ffmpeg（音声抽出）
- openai-whisper（文字起こし）
- python-docx（Word出力）

## インストール確認・自動実行
各ライブラリが未インストールの場合は処理前に自動インストールを提案し、
ユーザーの承認後にインストールして続行すること。

## スキルの目的
AI活用提案書を Vite + React + Tailwind CSS で構築した

- フロントエンド：Vite + React 18 + TypeScript

10. **リスクと対策** - リスクマトリクス表示

## スキルの目的
URL が不明な状態から Web 検索を行い、
通常の検索エンジンと異なり各ページの全文テキストも同時取得する。

## 使用 API
Firecrawl API の `/v1/search` エンドポイント

## 入力
- 検索クエリ（引数として受け取る）

1. 環境変数 `FIRECRAWL_API_KEY` を確認
2. Firecrawl `/v1/search` に POST
   ```json
   {
     "query": "{検索クエリ}",
     "limit": 5,
     "lang": "ja",
     "scrapeOptions": { "formats": ["markdown"] }
   }
   ```
3. 結果を整形して表示
   - タイトル・URL・全文Markdown を構造化して出力
4. ユーザーが特定URLの詳細を求めた場合は /firecrawl-scrape に引き継ぐ

## 出力フォーマット
```markdown
## 検索結果：{クエリ}

### 1. {タイトル}
URL: {url}
概要: {description}

{全文Markdownの冒頭500文字}

---
```
