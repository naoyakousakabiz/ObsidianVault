# 04_AI コンテキスト

## 役割
AI戦略・自動化設計・実装ナレッジの正本管理。このVault全体のAI活用を統括するフォルダ。

## フォルダ構成
| フォルダ | 内容 |
|---|---|
| `01_ツール運用/` | AI戦略・ツール役割・運用設計の正本 |
| `02_` | 欠番（将来の拡張用に確保） |
| `03_手法/` | 読書術・ファクトチェック等の手法ナレッジ |
| `04_学習/` | 外部講座・セミナー参照資料 |
| `80_実験ログ/` | 短期実行計画・一時メモ |
| `99_アーカイブ/` | 旧版・Deprecated資料 |

## 正本ファイル（迷ったらここを参照）
| ファイル | 内容 |
|---|---|
| `01_ツール運用/01_AI活用整理.md` | ツール役割・運用設計 |
| `01_ツール運用/02_AIキャッチアップ.md` | Phase・ガバナンス・品質ゲート |
| `01_ツール運用/03_NotebookLM運用.md` | NotebookLM構成・運用 |
| `01_ツール運用/05_データ蓄積ガイド.md` | 各フォルダへの蓄積データ一覧 |

## このフォルダでやること・やらないこと
- やること：AI戦略の意思決定、ツール設計、Skill/Hook/MCP設定、実装ナレッジ
- やらないこと：各ドメインの業務自動化設計（→各ドメインの`AI自動化/`フォルダ）、実績数値（→`03_Career/`）

## Claude Codeへの指示方針
- 設計変更時は必ず正本ファイルを更新すること（複数箇所に書かない）
- 自動化設計はドメインフォルダ配下に置き、このフォルダには戦略・設計思想のみ残すこと
- 実装ログは「作ったもの・使った技術・成果・改善点」をセットで記録すること
