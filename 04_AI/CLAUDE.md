# 04_AI コンテキスト

## 役割
AI戦略・自動化設計・実装ナレッジの正本管理。このVault全体のAI活用を統括するフォルダ。

## フォルダ構成
| フォルダ | 内容 |
|---|---|
| `01_運用/` | AI戦略・ツール役割・運用の正本（**直下 7 本** `00` `01` `02` `03` `05` `06` `10`。旧 `04`§6・実行ハブは `03_学習/01_実行計画.md` へ移動。旧 `07`→`01`付録A、旧 `09`→`99_アーカイブ`。整理は `00_索引.md`） |
| `04_テンプレート/` | **汎用テンプレ正本**（AI導入コンサル。司令塔 `01_AI導入ロードマップ.md`。各ドメイン `50_業務自動化/` のコピー起点） |
| `02_手法/` | **再現可能手順のみ**を置く（運用ルール: `02_手法/00_運用ルール.md`） |
| `03_学習/` | 外部講座・セミナー・要約の**未昇格キュー**＋実行ハブ `01_実行計画.md`（運用ルール: `03_学習/00_運用ルール.md`） |
| `05_SNSノウハウ/` | note・X 等 SNS／コンテンツ配信の型・検証メモ（AI手順が主のもの） |
| `99_アーカイブ/` | 旧版・Deprecated資料 |

## 正本ファイル

**パス付き一覧の正本は `README.md`（本フォルダ直下）の「正本ファイル（更新先）」。** ここでは表を置かない。

- **ツール運用フォルダの入口・読み順:** `01_運用/00_索引.md`
- **30日講座×Phase×直近・継続（蓄積・ルーティン）の単一窓口:** `03_学習/01_実行計画.md`
- **迷ったら:** 上記 `00_索引` のMECE表または `README.md`「利用シーン」

## このフォルダでやること・やらないこと
- やること：AI戦略の意思決定、ツール設計、Skill/Hook/MCP設定、実装ナレッジ
- やらないこと：各ドメイン**固有**の業務自動化実体・フロー（→各ドメインの `50_業務自動化/` 等）、実績数値（→`03_Career/`）

## Claude Codeへの指示方針
- 設計変更時は必ず正本ファイルを更新すること（複数箇所に書かない）
- **汎用テンプレ**は `04_テンプレート/` に置く。**ドメイン固有**の設計・施策は各 `50_業務自動化/` に置く
- 実装ログは「作ったもの・使った技術・成果・改善点」をセットで記録すること
