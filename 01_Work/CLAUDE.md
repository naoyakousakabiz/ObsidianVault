# 01_Work コンテキスト

## 役割
本業（SaaS企業・Sales/CS職）の実務ナレッジとスキル蓄積。「転職しても持ち運べる資産」として構造化する。

## オーナー情報（仮・転職後に更新）
- 職種：SaaS企業のSales / Customer Success（予定）
- 方向性：エンタープライズ営業またはCSマネジメント
- ※ 入社後に会社名・具体的な役割・チーム構成を追記すること

## フォルダ構成
| フォルダ | 内容 |
|---|---|
| `10_スキル/` | 転職しても持ち運べるスキル（Sales/CS/共通） |
| `20_プレイブック/` | 業務フロー・テンプレ・再現性ある手順 |
| `30_プロジェクト/` | 社内プロジェクト記録（匿名化） |
| `40_マネジメント/` | チーム・組織管理（管理職時に使用） |
| `50_AI自動化/` | 業務自動化の設計・実装（Work領域） |
| `80_メモ/` | 会議メモ・1on1ログ（一時置き場） |

## このフォルダでやること・やらないこと
- やること：スキルナレッジの蓄積、業務フロー設計、会議メモ・1on1ログ
- やらないこと：キャリア戦略・転職管理（→`03_Career/`）、個人事業（→`03_Business/`）、AI全体戦略（→`04_AI/`）

## データ原則
- クライアント固有情報・個人情報は書かない（必ず匿名化）
- `80_メモ/` のメモは週次で `10_スキル/` か `20_プレイブック/` へ昇格判断する
- 「この会社でしか通じない情報」より「次の会社でも使えるナレッジ」を優先する

## Claude Codeへの指示方針
- スキル整理時は「この経験が面接でどう語れるか」の視点も加えること
- 商談・CS対応パターンは匿名化して汎用的なフレームワークとして抽出すること
- 自動化設計が必要な場合は `01_Work/50_AI自動化/` に設計を置くこと
