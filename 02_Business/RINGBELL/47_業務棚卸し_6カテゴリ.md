# RINGBELL 業務棚卸し（6カテゴリ）

**目的:** 副業 RINGBELL（婚活CS・担当コンシェルジュ観点）の業務を、効率化・自動化の議論にそのまま載せられる粒度で棚卸しする。

**正本:** `10_CS業務フロー分解.md`（フロー・業務カード・R&R・SLA）。本書は同正本に**のみ**アンカーする。転職・個人メディアなどは**含めない**。

---

## 1. カテゴリの切り方（このフォルダ内での定義）

| カテゴリ | RINGBELL における含意 |
|:--|:--|
| **メール関連** | **非同期メッセージ**全般。業務分解上は **Gmail**（決済通知確認）・**公式LINE**（会員）・**Slack**（社内・事務・営業・統括）が主。 |
| **会議関連** | **同期の対話**（Meet 面談、週次定例）と、直前・直後のオペ（リマインド、録画、共有）。 |
| **書類作成** | **再利用・監査可能なテキスト・台帳**（テンプレ、PR 文、Slack 投稿体裁、スプレッドシート入力、事前フォーム設計は「入力物」扱いでここに寄せる）。 |
| **SNS運用** | **不特定多数向けのチャネル運用**（X / YouTube / note 等）。**業務分解の担当コンシェルジュ範囲には定義なし**（下記 §5 参照）。 |
| **経理・会計** | **会員の月会費・決済の確認オペ**、例外時の事務連絡。自宅側の確定申告等は本書のスコープ外だが §7 で1行触れる。 |
| **データ分析** | **会員DB・SLA・目標/実績差分**の把握と、改善への接続（集計・可視化・解釈）。 |

**MECE メモ:** 会員向け **公式LINE** は「SNS運用」ではなく **メール関連（非同期CS）** に置く。

## 5. SNS運用業務

**業務分解正本のスコープ:** 担当コンシェルジュの主軸は §11.1 のとおり **チャット返信**と**定期面談**。**X / YouTube / note 等の不特定多数向け SNS 運用は定義されていない**（企業アカウント運用が別 R&R なら本ノートの外）。

| 項目 | 内容 |
|:--|:--|
| **本カテゴリ（RINGBELL 委託 CS）** | 棚上げのみ。**実務は §2 の公式LINE に集約**。 |
| **個人の婚活情報発信**を別レーンで行う場合 | Vault 全体の話としては `CLAUDE.md` のプロフィール／個人目標に接続。本ファイルでは**扱わない**。 |
