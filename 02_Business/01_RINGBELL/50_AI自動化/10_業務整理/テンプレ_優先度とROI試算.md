# テンプレ：優先度とROI試算（個人・小規模向け）

> **使い方:** ボトルネックや施策候補を列挙し、どれから着手するかを決める。  
> 数字が出ない項目は「大/中/小」の感覚値でよい。精度より**「議論できる形」にすること**が目的。

## 1. 施策候補マトリクス

| # | 施策・改善候補 | 解決するボトルネック | 実装工数 (S/M/L) | 効き (小/中/大) | 優先度 | 判断理由 |
|:--|:--|:--|:--|:--|:--|:--|
| 1 | 例：返信文の下書き自動生成 | 手書き返信に1通15分 | S | 大 | 1 | 繰り返し頻度が高く即効性あり |
| 2 |  |  |  |  |  |  |
| 3 |  |  |  |  |  |  |

**工数の目安:**
- S = 数時間〜1日（プロンプト調整＋既存ツール連携）
- M = 数日〜1週間（API連携・フロー構築）
- L = 2週間以上（システム改修・外部依存あり）

**効きの目安（いずれかで判断）:**
- 大 = 月10時間超の削減 / ミス率半減以上 / 顧客体験の可視的な改善
- 中 = 月3〜10時間 / 部分的な改善
- 小 = 月3時間未満 / 補助的

---

## 2. 優先度の決め方（ルール）

> **AIありきにしない。** 工数が小さく効きが大きいものを先に着手し、余裕ができたら大きい施策へ。

1. **効き「大」×工数「S/M」を最優先**（Quick Win）
2. **効き「大」×工数「L」は PoC 1本に絞ってから拡大**
3. **効き「小」はAI以外の業務改善で閉じられないか先に確認**（ロードマップ #11）

---

## 3. ROI仮説（ざっくりでよい）

| 施策 | 月コスト（ツール料＋工数） | 月の効き（時間×単価 or 件数） | 回収の目安 |
|:--|:--|:--|:--|
| 例：返信下書き | APIコスト 〜500円/月 | 15分×60件=15h=約1.5万円相当 | 初月から黒字 |
|  |  |  |  |

> **コストが不明なときは「上限いくらまでなら使うか」だけ決めておく（撤退条件と紐づける）。**
