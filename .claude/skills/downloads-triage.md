# /downloads-triage — Downloads の整理

## これは何か

`~/Downloads` など、溜まったファイルを**拡張子・日付・用途**で分けるルールと、**移動・リネームの手順**を作るスキルです。**削除は実行しません**。候補を出して、実行は本人の判断に任せます。

## いつ使うか

- ダウンロードフォルダが溢れて、探すのがしんどいとき
- アーカイブの**ルール**をはっきり決めたいとき

## 使わないほうがよい場合

- ゴミ箱を空にしたいだけ、完全自動で消したいとき（このスキルの範囲外）
- すでに専用ツールで運用が固まっているとき

## 前提条件

- ターミナルや Finder で**一覧や移動ができる**こと。コマンド案は出すが、**実行はユーザー**。

## どう依頼するか

1. **`downloads-triage`** と対象パス（省略時は `~/Downloads` を想定）。
2. **絶対に触らない**フォルダや名前のパターン。
3. 移動先の親フォルダや分類の考え方。

**例:** 「`downloads-triage`。`~/Downloads`。`Invoices/` 以下は触らない。プロジェクト別に `~/Archive/年`。」

## 進め方（エージェント向け）

1. 件数や主な拡張子など、現状をユーザーと確認する。
2. ルール案を出し、**合意してから**手順に落とす。
3. 移動前に、**何件くらい動くか**を見積もる。
4. **削除コマンドは出さない**。削除候補はリストだけ。
5. クラウド同期がある場合は負荷に一言触れる。

## 完了の目安

- ルールを**一文で説明**できるほど単純
- 除外パスがはっきり書いてある
- 削除は**提案リストのみ**で終わっている
