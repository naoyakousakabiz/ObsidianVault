# /generate-article-images — 記事用のアイキャッチ

## これは何か

ブログや note などで、**見出しごとにアイキャッチ**を揃えたいとき向けです。章ごとの**画像プロンプト**と alt 文案を出し、API が使えれば生成手順も。トーンと**出してはいけないもの**を守ります。

## いつ使うか

- 公開記事で、章ごとにビジュアルを揃えたいとき
- チームで**プロンプトの型**を共通にしたいとき

## 使わないほうがよい場合

- **求人用のバナー1枚**が主目的 → `job-eyecatch`
- 画像は不要で構成レビューだけのとき

## 前提条件

- 画像 API を使う場合は**キーはユーザー環境**。無い場合はプロンプトと alt だけで完結。

## どう依頼するか

1. **`generate-article-images`** と見出し一覧または本文。
2. 画風、アスペクト比、**NG なモチーフ**。
3. 使う生成エンジンが決まっていれば書く。

**例:** 「`generate-article-images`。H2 が5つ。横長 1200×630。実在ロゴと実在の顔は避ける。落ち着いたビジネストーン。」

## 進め方（エージェント向け）

1. トーンと色数・構図の方針を 1 文で固定する。
2. 章ごとに**プロンプト・代替案1・alt** を出す。
3. 商標や著名人の**そっくり表現**を避ける注意を添える。
4. 商用可否はユーザーがサービス利用規約で確認する旨を書く。

## 完了の目安

- 見出しぶん**画風がそろっている**
- alt が付き、アクセシビリティの最低限を満たす
- 禁止モチーフに触れていない
