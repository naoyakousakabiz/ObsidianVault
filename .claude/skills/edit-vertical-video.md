# /edit-vertical-video — 縦動画（ショート）の自動編集 v2

## これは何か

TikTok / Shorts / Reels 向け縦動画を **1コマンドで自動編集**するスキル。

| ステップ | 処理 |
|---|---|
| 1 | **自動クロップ**（横動画→縦9:16）＋明るさ・コントラスト補正 |
| 2 | **無音カット**（0.3秒以上の無音を除去） |
| 3 | **Whisper文字起こし**（通常 or カラオケ風ワードハイライト） |
| 4 | **Claude API字幕チェック**（誤認識漢字・不自然語句を自動修正） |
| 5 | **字幕焼き込み**（シンプル底部配置）＋冒頭タイトル＋フェードイン/アウト |
| 6 | **BGM合成**（サイドチェイン圧縮でボイス優先ダッキング） |
| 7 | **尺カット**（プリセットの最大時間に合わせて自動トリム） |
| 8 | **サムネイル自動生成**（_thumb.jpg） |
| 9 | **ディレクターレビュー**（Claude APIでクオリティ評価・改善提案） |

出力: `{元ファイル名}_final.mp4` + `{元ファイル名}.srt` + `{元ファイル名}_thumb.jpg`

## スクリプトパス

```
99_System/scripts/vertical_video_pipeline.py
```

## 実行前の必須確認（不明な場合は必ずユーザーに質問する）

エージェントは実行前に以下を確認し、**不明な項目があれば一括で質問してから実行**する。

| 確認項目 | デフォルト | 質問が必要なケース |
|---|---|---|
| ファイル名 | — | **必須。不明なら必ず聞く** |
| 投稿プラットフォーム | tiktok | 言及がなければ「TikTok向けでいいですか？」と聞く |
| BGM | なし | 「BGMはつけますか？（pop/lofi/energetic/ファイル指定/なし）」と聞く |
| カラオケ字幕 | なし | 「字幕はカラオケ風（単語ハイライト）にしますか？」と聞く |
| 冒頭タイトル | なし | 「冒頭にタイトルテキストを入れますか？」と聞く |

**NG行動：** 情報が不足したままデフォルトで勝手に実行しない。

## エージェントの動き方（編集者＋ディレクター二段階）

### Phase 1: 編集者（スクリプト実行）

1. ファイル名を特定する（Downloadsフォルダを優先）
2. **確認項目を一括で質問する**（言及済みはスキップ）
3. 回答を受け取ってからスクリプトを実行：

```bash
python3 "99_System/scripts/vertical_video_pipeline.py" \
  "{ファイル名}" \
  --preset tiktok \
  [--bgm "{BGMパス}" or --download-bgm] \
  [--bgm-genre pop|lofi|energetic] \
  [--title "タイトルテキスト"] \
  [--karaoke] \
  [--model small|medium|large]
```

### Phase 2: ディレクター（Claude APIによるセルフチェック）

スクリプト完了後、エージェント自身が以下の観点で**厳しく**レビューし、**改善提案をユーザーに報告**する。問題があれば自動で再処理または修正案を提示すること。

#### 【A】字幕セルフチェック（最優先・厳格に実施）

SRTファイルを読み込み、以下を全件チェックする：

| チェック項目 | 基準 | 判定方法 |
|---|---|---|
| **1行の文字数** | `SUBTITLE_MAX_CHARS`（現在7文字）以内 | 各行の文字数をカウント |
| **最大行数** | 2行以内 | 改行数を確認 |
| **画面はみ出し推定** | 1行7文字 × フォント140px ≈ 980px → 1080px以内か | 文字数 × フォントサイズ で推算 |
| **誤字・誤認識** | 文脈上不自然な漢字・語句がないか | Claude APIで確認（APIキーがある場合） |
| **空白・記号の乱れ** | 不要な空白、文中の句読点ズレがないか | 目視確認 |
| **表示タイミング** | 字幕の尺（end - start）が短すぎないか（0.5秒以上） | タイムスタンプ差分を確認 |

**字幕チェックでNGがあった場合：**
- 文字数オーバー → `wrap_srt_file` を再実行するか、手動で分割案を提示
- はみ出し懸念 → `SUBTITLE_FONT_SIZE` を下げる提案をユーザーに伝える
- 誤認識 → 修正後のSRT内容を提示してユーザーに確認を求める

#### 【B】映像・音声チェック

1. **フック（冒頭3秒）**
   - 字幕の1行目は「続きが気になる」内容か？
   - 視聴者が離脱しないキーワードがあるか？

2. **ペーシング**
   - 無音カット後の尺は適切か？（長すぎないか）

3. **BGM**
   - BGMが入っているか
   - ボイスダッキングが機能しているか（声 > BGM）

4. **プラットフォーム適合**
   - 尺がプリセットの制限内か
   - 縦型（9:16）になっているか

**ディレクター報告フォーマット:**
```
【字幕チェック】
✅ 問題なし: 文字数・行数・はみ出し
⚠️ 改善推奨: [項目] → [具体的な提案]
❌ 要修正: [項目] → [修正方法]

【映像・音声チェック】
✅ / ⚠️ / ❌ [各項目]
```

## BGM音量について（重要）

- **サイドチェイン圧縮**を使用：声が出ている間はBGMを自動で下げ、無音時は戻す
- デフォルト基準音量: `--bgm-vol 0.18`（声に対してBGMは控えめ）
- BGMが大きすぎる場合: `--bgm-vol 0.08` に下げる
- BGMが小さすぎる場合: `--bgm-vol 0.25` に上げる
- ボイス側は `-14 LUFS` に正規化（SNS標準）

## 字幕スタイル仕様

- フォント: Hiragino Sans / Bold
- サイズ: 52px
- 白文字 + 黒縁（Outline=2）+ シャドウ
- 位置: 下部中央（MarginV=60）
- 背景ボックス: なし（シンプル）

## 動画編集クオリティ向上ノウハウ

### TikTok/Shorts で伸びる構成
1. **冒頭0-3秒**: 結論 or 衝撃的なワードで掴む（「〇〇って知ってた？」「これやばい」）
2. **3-10秒**: 問題提起・共感
3. **10秒〜**: 解決策・本題
4. **最後3秒**: CTA（「フォローして」「保存して」）

### 字幕のベストプラクティス
- 1行20文字以内（視認性）
- 強調ワードは別セグメントに分ける
- カラオケ字幕は英語コンテンツに特に有効

### BGMのベストプラクティス
- ボイスに対してBGMは-20dB以上小さく
- サビ冒頭に合わせると演出効果が上がる
- lofi: 落ち着いた解説系 / pop: 明るいTips系 / energetic: モチベ系

## プリセット一覧

| プリセット | 解像度 | fps | 最大尺 |
|---|---|---|---|
| `tiktok` | 1080×1920 | 30 | 60秒 |
| `reels` | 1080×1920 | 30 | 90秒 |
| `shorts` | 1080×1920 | 60 | 60秒 |
| `default` | 1080×1920 | 30 | 制限なし |

## 全オプション早見表

| オプション | デフォルト | 説明 |
|---|---|---|
| `--preset` | tiktok | プラットフォームプリセット |
| `--model` | medium | Whisperモデルサイズ |
| `--lang` | ja | 言語コード |
| `--karaoke` | false | カラオケ風ワードハイライト字幕 |
| `--title` | なし | 冒頭3秒に表示するタイトル |
| `--no-enhance` | false | 明るさ・コントラスト補正をスキップ |
| `--download-bgm` | false | BGMを自動ダウンロード |
| `--bgm` | なし | BGMファイルパスを直接指定 |
| `--bgm-genre` | pop | pop / lofi / energetic |
| `--bgm-vol` | 0.18 | BGM基準音量（ダッキングで自動調整） |
| `--silence-db` | -35 | 無音判定閾値dB |
| `--silence-sec` | 0.5 | 無音最小秒数 |
| `--output-dir` | 入力と同じ | 出力先ディレクトリ |

## Claude API字幕チェックについて

- `ANTHROPIC_API_KEY` が環境変数にあれば自動実行
- セット方法: `~/.zshrc` に `export ANTHROPIC_API_KEY=sk-ant-...` を追加

## BGMキャッシュ

- 初回DL後は `~/Downloads/_bgm_cache/` にキャッシュされ、次回から再利用

## 注意事項

- Whisperモデル初回DL: medium≒1.5GB
- 処理時間の目安: 5分動画で4〜6分
- ディスク空き容量は元動画の3倍以上を確保
