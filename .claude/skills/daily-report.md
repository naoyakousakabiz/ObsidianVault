# /daily-report — 今日の作業日報を `80_Journal/01_Daily/YYYY-MM-DD.md` に追記

## 目的

Cursor/Claude Code 経由で行った作業を「後から振り返れる形」で **日報として固定**する。

## 保存先（原則）

- `80_Journal/01_Daily/YYYY-MM-DD.md`
  - 既に存在する場合は **上書きしない**
  - 末尾に追記（追記ブロックが既にあれば、そのブロックだけ更新）

## ルール（重要）

- **TODO 管理（進行中の更新）はここではしない**（日報＝確定ログ）。
- 機微情報・クライアント固有情報は書かない（匿名化）。
- 推測で「やったこと」を捏造しない。根拠（差分・ファイル更新・コマンド出力）を優先。

## 入力

- 引数なし: 今日（ローカル日付）
- 引数あり: `YYYY-MM-DD`（例: `/daily-report 2026-04-09`）

## 収集（エージェントのやること）

1. 対象日付を確定し、`80_Journal/01_Daily/YYYY-MM-DD.md` を読む。
2. 可能なら git 情報を集める（このVaultがgit repoの前提）:
   - `git status`
   - `git diff`
   - `git log -n 10 --oneline`
3. 可能なら「今日更新されたファイル」を拾う（大雑把でOK）:
   - `ls -lt`（Vault 内の主要ディレクトリ。必要最小限）
4. 直近のターミナル実行ログがあれば、要点だけ抽出（コマンド名レベルで十分）。

## 出力フォーマット（追記ブロック）

`80_Journal/01_Daily/YYYY-MM-DD.md` の末尾に、次を追記する（必ずこの見出しを使う）。

### 日報（自動）

- **今日やったこと（確定）**
  - （箇条書き）
- **成果物 / 変更**
  - （作成/更新したファイル、設定、スキル、コマンド等。Vault 内パスは書く）
- **詰まり / 学び**
  - （1〜3個）
- **次にやること（提案）**
  - （最大3つ。TODO管理とは分離し「提案」として書く）
