# /talmood-invoice — 請求フロー（freee ＋ Sheets 想定）

## これは何か

請求先・品目・期日・税区分を整理し、**freee** での登録の流れと、**Google Sheets** での入金管理などの更新案をセットで出すスキルです。**API のシークレットは扱いません**。

## いつ使うか

- 請求から入金確認までを**型にはめたい**とき
- freee に載せる前に、**行項目の下書き**が欲しいとき

## 使わないほうがよい場合

- 税務・契約の**最終判断だけ**が知りたいとき（税理士・正本が先）
- 請求と無関係な記帳だけ

## 前提条件

- freee の事業所など、請求に必要な設定が済んでいること。**トークンは会話に貼らない**。

## どう依頼するか

1. **`talmood-invoice`** と相手・案件・金額の内訳・希望の請求日。
2. freee は API か画面操作か。
3. Sheets の列名やシートのルール。

**例:** 「`talmood-invoice`。株式会社ダミー向け。品目 2 つ・税込。freee は画面想定。入金シートは A 列を請求番号に。」

## 進め方（エージェント向け）

1. 税区分と端数ルールを確認する。
2. freee 側は**チェックリスト**（取引登録→請求書発行など）にする。
3. Sheets は**請求番号など一意のキー**を必ず含める設計にする。
4. 自動化するなら、失敗時のやり直し方に触れる。

## 完了の目安

- インボイス制度や税について**要確認**が書いてある
- 秘密をユーザーに**貼らせようとしていない**
- 送付メールは**人が確かめてから**と明記されている
