# /sf-daily-report — Salesforce の日次スナップショット

## これは何か

商談やパイプラインなどを、決めた期間と項目で集計し、**日付つきの読み取り用スナップショット**（表と短い文章）にします。CRM への書き込みは**別途・合意のうえ**で。

## いつ使うか

- 朝会などで**数字の推移**を短く説明したいとき
- 標準ダッシュボードにない切り口を、**しばらくだけ**追いたいとき

## 使わないほうがよい場合

- Notion 顧客マスタとの突き合わせが主 → `sync-client-registry`
- **レコード更新や一括変更**が主目的のとき（別プロセス）

## 前提条件

- 参照するオブジェクト・フィールドを**読める**こと。

## どう依頼するか

1. **`sf-daily-report`** と、欲しい指標・期間・集計の単位。
2. データの取り方（画面・API・すでにある CSV）。
3. 社名などをマスクするか。

**例:** 「`sf-daily-report`。Stage 別の件数と金額。期間は今四半期。会社名はマスク。」

## 進め方（エージェント向け）

1. 期間の境界とタイムゾーン、会計年度の考え方を固定する。
2. **スナップショット日時**を明記する。前日比が要れば列を足す。
3. 更新操作が要る場合はリスクを説明し、**別タスク**にすると明示する。
4. 個人を特定する ID は最小限にする。

## 完了の目安

- 指標の定義が**短く言える**
- 正本は Salesforce とし、ここは**参照用**と分かる
- API 制限やバルク処理に一言触れている
