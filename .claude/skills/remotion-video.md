# /remotion-video — Remotion で解説動画

## これは何か

台本とデザインのルールから **Remotion の構成**（コンポジション分け）とレンダリング手順を提案し、**同じ型の動画を何本も**出しやすくするスキルです。

## いつ使うか

- 同じ型の解説を繰り返し出したいとき
- 図やテロップを**コードで管理**したいとき

## 使わないほうがよい場合

- 1 本だけの短い切り抜きや実写メイン → `edit-vertical-video` の方が早いことがある
- Remotion を入れたくない PoC

## 前提条件

- Node が動き、Remotion を入れられること。初回は依存とレンダ時間に余裕を。

## どう依頼するか

1. **`remotion-video`** と章立てまたは台本、目標の尺・解像度・fps。
2. フォント・色・ロゴのルール。既存プロジェクトのパス。
3. 納品形式（MP4、解像度のバリエーションなど）。

**例:** 「`remotion-video`。`./video` に既存。16:9、1080p60。イントロ・本編3ブロック・アウトロで計 90 秒。ブランド色 #1a2b3c。」

## 進め方（エージェント向け）

1. Composition の分け方と、**props や JSON** で渡すデータを決める。
2. 小さな UI パーツに分けて再利用しやすくする。
3. `remotion` CLI のプレビュー・レンダコマンドを環境に合わせて書く。
4. 長いときはセグメントレンダや並列数に触れる。

## 完了の目安

- フォルダ構成とデータの流れが**短く説明できる**
- レンダ手順がその環境で**動きそうなコマンド**になっている
- フォント・素材のライセンスをユーザーが確認する旨がある
