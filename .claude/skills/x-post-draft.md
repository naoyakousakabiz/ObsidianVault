# /x-post-draft — X 投稿文案 3 案

## これは何か

**トピックを渡すと、140 字以内の X（Twitter）投稿文を 3 パターン**出す。
各パターンに**ハッシュタグ提案**を添える。**手動投稿**は本人確認後。**Typefully API で予約**する場合は `/typefully-schedule`。

## いつ使うか

- ブログ・note・ウェビナー・サービスの告知文を複数パターン比べたいとき
- SNS 投稿に時間をかけたくないが、品質は担保したいとき
- A/B で反応を試したいとき

## 使わないほうがよい場合

- 長文スレッド（複数ツイート連投）が目的 → 別途構成を相談
- 画像プロンプトが主目的 → `generate-article-images` / `job-eyecatch`
- 顧客や会員向けの**公式アカウントからの即時送信**が目的 → 承認フローを必ず挟む

## 前提条件

- トピックのキーワードまたは一文を渡す（例: 「AIで業務効率化した話」）
- 任意: トーン（カジュアル/ビジネス/共感型など）、ターゲット読者、禁止ワード

## どう依頼するか

1. **`x-post-draft`** とトピック。
2. 必要なら「カジュアルに」「BtoB向け」「URLは含めない」など条件を追加。
3. ハッシュタグ案の数・言語（日本語のみ／英語混在）を指定してもよい。

**例 1（最小）:** 「`x-post-draft`。トピック: 『Claude Code で Vault の自動化を始めた』」  
**例 2（条件あり）:** 「`x-post-draft`。トピック: AI活用で営業成績が上がった体験談。カジュアル＆共感型。BtoB 営業パーソン向け。ハッシュタグは日本語3つ以内。」

## 進め方（エージェント向け）

1. トピックから**訴求軸**を 3 つ選ぶ（例: 驚き・共感・問いかけ）。
2. 各軸で投稿文を 1 本ずつ書く。**文字数を必ずカウント**し、140 字以内に収める。
3. 各案に**ハッシュタグ 2〜4 個**を提案する（日本語優先、必要なら英語混在）。
4. 3 案を比較しやすいよう**表形式**でまとめる。
5. URLを含む場合は「短縮 URL で約 23 字消費」と注記する。
6. **絵文字**は指定がなければ各案 0〜1 個にとどめる。

### 出力フォーマット例

| # | 訴求軸 | 本文（140字以内） | 字数 | ハッシュタグ案 |
|---|--------|-------------------|------|----------------|
| 1 | 驚き   | ...               | 〇〇 | #AI活用 #業務効率化 #Claude |
| 2 | 共感   | ...               | 〇〇 | #営業 #SaaS #AI |
| 3 | 問いかけ | ...             | 〇〇 | #自動化 #生産性 #AIツール |

## 完了の目安

- 3 案すべて**140 字以内**（URL含む場合は 23 字換算で調整済み）
- 訴求軸がそれぞれ異なる（3 案が似通っていない）
- ハッシュタグが各案に付いている
- 個人・顧客を特定できる表現を含まない
- 投稿の実行はユーザーが行う旨を添えた
