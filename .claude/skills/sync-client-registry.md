# /sync-client-registry — Notion の顧客 DB とローカル一覧

## これは何か

手元の CSV などと **Notion の顧客データベース**を、**どのプロパティと結びつけるか**、差分はどうなるか、という**同期プラン**を作るスキルです。値が食い違ったときは、**先にルールを決めてから**反映案を出します。

## いつ使うか

- フォルダ運用と Notion の**名寄せがズレた**とき
- 新しいプロパティを一括で埋めたい設計をしたいとき

## 使わないほうがよい場合

- Salesforce の数値スナップだけ欲しい → `sf-daily-report`
- Notion を使っていない運用のとき

## 前提条件

- Notion API や MCP が使えること。**インテグレーションのトークンは会話に出さない**。

## どう依頼するか

1. **`sync-client-registry`** とローカル側（パスや列名）。
2. Notion のデータベース ID と、対応させたい**プロパティ名**（分かる範囲）。
3. 食い違ったとき、**Notion 優先かローカル優先か**（列ごとなら列ごと）。

**例:** 「`sync-client-registry`。`clients.csv` の `code` を Notion の `Code` に。更新はローカル優先。新規ページだけ Notion に作る案が欲しい。」

## 進め方（エージェント向け）

1. ローカルのキーとページ ID の**対応表**の案を出す。
2. 新規・更新・アーカイブ候補に分け、**件数**を先に報告する。
3. 競合する列は一覧にし、**ユーザー合意後**に反映案を書く。
4. 更新は**小さなバッチ**に分け、途中で止まっても再開できるようにする。

## 完了の目安

- **競合時のルール**が一文である
- 影響件数が見える
- アーカイブ・削除を**勝手に実行する前提になっていない**
