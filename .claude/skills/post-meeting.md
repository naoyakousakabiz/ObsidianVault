    # /post-meeting — 会議後の自動化

    **やること:** 文字起こしから議事録・連絡・社内共有まで一気通貫（チャエン版フロー）

    **依存:** gogcli, Google Calendar, Notion MCP, Slack MCP

    ## 実行手順
    1. 文字起こしまたは要点をユーザーから受け取る（`/lf` でファイルでも可）。
2. 会議のクライアント名・日時・参加者を確定（不明なら質問）。
3. 議事録（.md）を作成。必要なら .docx 化手順を案内。
4. 御礼メール下書き・Slack 投稿文案・Notion DB 用プロパティ案を生成。
5. MCP が利用可能なら各サービスへ反映。不可ならコピペ用ブロックだけ渡す。

    ## Vault での共通ルール
    - クライアント固有情報・実名は扱わない（匿名化）。
    - `clients/` 等の例パスは、このリポジトリの慣習に**置換**する。議事録・アウトプットの置き場はユーザーに確認する。
    - MCP / API が無い操作は**テキスト案・手順だけ**出し、実行はユーザーが行う。
