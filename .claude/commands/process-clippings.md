`20_Input/02_Clippings/` にある未処理のWebクリッピングを読み込み、洞察・要点に変換してドメインフォルダへ転記する。

**引数:** $ARGUMENTS（省略可。処理対象のキーワードを絞る場合に指定。例: `AI Claude`）

## 実行手順

1. `20_Input/02_Clippings/_processed_log.md` を読み、処理済みファイルリストを把握する
2. `20_Input/02_Clippings/` 内の `.md` ファイルのうち、未処理のものを列挙する
   - 引数が指定された場合はファイル名・内容に含むものだけを対象にする
3. 各クリッピングについて以下を行う：
   a. 記事の主題とキーポイントを把握する
   b. 自分のVaultのどのドメインに関係するかを判定する（下記マッピング参照）
   c. 「洞察・アクション・保存価値」を判定する（保存価値がない場合はスキップ可）
   d. 変換した内容を該当ドメインフォルダに保存する、または `00_Inbox/` に仮置きしてユーザーに判断を仰ぐ
4. `_processed_log.md` に処理済みとして記録する

## ドメインマッピング

| 記事ジャンル | 転記先 |
|---|---|
| AI / Claude Code / 自動化 | `04_AI/02_手法/` または `04_AI/03_学習/` |
| SNS / note / コンテンツ配信 | `04_AI/05_SNSノウハウ/` |
| Sales / CS / SaaS / ビジネス | `01_Work/10_スキル/` 該当サブフォルダ |
| キャリア / 転職市場 | `03_Career/30_キャリア戦略/30_synthesis/` |
| 健康 / トレーニング / 栄養 | `09_Athlete/` 該当サブフォルダ |
| 美容 / スキンケア | `11_Beauty/` |
| 投資 / 資産 / ライフプラン | `05_Life/30_finance/` |
| 文化 / 教養（分類困難） | `10_Culture/` |
| 判断困難 | `00_Inbox/` に仮置き |

## 出力フォーマット（転記先ファイル）

```markdown
---
date: YYYY-MM-DD
type: clipping
domain: [ドメイン名]
status: active
source: claude-code
generated_at: YYYY-MM-DD
reviewed: false
tags: [clipping]
---

# 【記事タイトル】

**元URL：**
**クリップ日：**

## 要点（3行以内）

## 自分のVaultへの示唆

- 

## アクション候補

1. 

```

## ルール

- 記事の要約にとどめず「自分のVaultでどう使えるか」の観点で整理する
- 保存価値が低い記事はスキップしてよい（`_processed_log.md` にスキップ理由を一行残す）
- 元ファイル（`20_Input/02_Clippings/` の `.md`）は削除しない
- `_processed_log.md` への記録を必ず行う
