`20_Input/01_Kindle/` にある未処理のKindleハイライトを読み込み、洞察・アクションに変換してドメインフォルダへ転記する。

**引数:** $ARGUMENTS（省略可。処理対象の書籍名を絞る場合に指定。例: `カスタマーサクセス`）

## 実行手順

1. `20_Input/01_Kindle/_processed_log.md` を読み、処理済み書籍リストを把握する
2. `20_Input/01_Kindle/` 内の `.md` ファイルのうち、未処理のものを列挙する
   - 引数が指定された場合はファイル名に含むものだけを対象にする
3. 各書籍について以下を行う：
   a. ハイライト全体を読み、書籍テーマと主要メッセージを把握する
   b. 自分のVaultのどのドメインに関係するかを判定する（下記マッピング参照）
   c. ハイライトを「学び・実践アクション・参照メモ」に変換する（ハイライト転記にとどめない）
   d. 変換した内容を該当ドメインフォルダに保存する
4. `_processed_log.md` に処理済みとして記録する（書籍名・処理日・転記先）

## ドメインマッピング

| 書籍ジャンル | 転記先 |
|---|---|
| Sales / CS / SaaS / ビジネス | `01_Work/10_スキル/` 該当サブフォルダ |
| キャリア / 転職 / 自己分析 | `03_Career/30_キャリア戦略/` |
| AI / テクノロジー / 自動化 | `04_AI/02_手法/` |
| 健康 / 競技 / トレーニング | `09_Athlete/` 該当サブフォルダ |
| 投資 / 資産 / ライフプラン | `05_Life/30_finance/` |
| 人間関係 / 心理 | `08_Relationship/50_patterns/` |
| 文化 / 教養（分類困難） | `10_Culture/` |
| 複数ドメインにまたがる | メインドメインに置き、関連を末尾に記載 |

## 出力フォーマット（転記先ファイル）

```markdown
---
date: YYYY-MM-DD
type: reading-log
domain: [ドメイン名]
status: active
source: claude-code
generated_at: YYYY-MM-DD
reviewed: false
tags: [書籍, kindle]
---

# 【書籍名】読書ログ

**著者：**
**読了・処理日：** YYYY-MM-DD

## 核心メッセージ（1〜3行）

## 学び（再利用できる形で）

- 

## 実践アクション

1. 

## 参照メモ（気になるハイライト・補足）

> 

```

## ルール

- ハイライトをそのまま転記しない。「自分のVaultでどう使えるか」に変換する
- 1書籍につき転記先は原則1ファイル（複数ドメインにまたがる場合はメインドメインに置く）
- 処理後は元ファイル（`20_Input/01_Kindle/` の `.md`）を削除しない（NotebookLMソースとして保持）
- `_processed_log.md` への記録を必ず行う
