面接前の想定Q&Aを生成し、`10_qa.md` を埋める。

**引数:** $ARGUMENTS
（形式: `<スラッグ> [面接段階: 1次/2次/役員]`、省略時は1次想定）

## 実行手順

1. 引数を読み取る
2. 以下を読む:
   - `03_Career/10_プロフィール/20_work_history.md`（実績・根拠の正本）
   - `03_Career/10_プロフィール/30_traits.md`（強み・弱み・志向）
   - `03_Career/40_転職活動/10_campaigns/10_202604_career_transition/companies/<スラッグ>/10_company_info.md`（企業情報・志望動機）
   - `03_Career/40_転職活動/10_campaigns/10_202604_career_transition/30_prep_common.md`（汎用トーク・既存の自己PR）
   - `03_Career/40_転職活動/10_campaigns/10_202604_career_transition/20_interviews.md`（よく聞かれるパターン）
3. 面接段階に応じた想定Q&Aを生成（最低15問）:

   | カテゴリ | 問数 |
   |---------|------|
   | 自己PR・転職理由・志望動機 | 3問 |
   | 実績・経験の深掘り | 5問 |
   | 企業固有（JD要件・事業理解） | 3問 |
   | 弱み・失敗・課題 | 2問 |
   | 逆質問 | 3問 |

4. 各Qに対して以下を埋める:
   - 回答の要点（箇条書き）
   - 根拠（`10_プロフィール/` の具体的な見出し・案件名）
   - 深掘りされそうな追問
   - 追問への一言回答

5. `10_qa.md` の「想定Q&A」テーブルに追記（既存行は残す、重複は統合）

## ルール

- 回答根拠は必ず `10_プロフィール/` の実在する実績・事実から引く
- 捏造・誇張なし。根拠がない主張は「（要補強: ○○に実績を追加してから使う）」と明示
- 企業固有の質問は `10_company_info.md` の調査内容から作る（一般論にしない）
- 弱みの回答は「認識 → 対策 → 成長」の構造で
