# NotebookLM Personal OS – 構成設計書（最終版）
### ── 30代男性の人生を最大化する11冊設計 ──

正本ファイル名：`03_NotebookLM運用.md`（旧：`AI戦略_04_NotebookLM運用.md`）

---

## 1. Purpose（目的）

本システムは、
「仕事・資産・身体・人生・創造・思考」を分離し、
各領域の意思決定を最適化することで、
個人パフォーマンスと人生全体の質を最大化する

---

## 2. Design Principles（設計原則）

- すべてのノートは「意思決定のため」に存在する
- 構造はシンプルに保つ
- 実態ベースで分ける（概念で分けない）
- 将来拡張は「後から分割」で対応する

---

## 3. Notebook Structure（全体構成・11冊）


## 4. 各ノートブック詳細設計

---

### 01 Career
**定義：** 雇用キャリアの意思決定

**活用目的：**
会社員としての市場価値を最大化し、昇進・転職・副業シフトなど
キャリアの分岐点で最適な判断を下す

**ソース例：**
```
・職務経歴書・スキルマップ
・業界トレンド記事（定期追加）
・1on1・評価面談のメモ（匿名化）
・目指すロールモデルの情報
・転職市場の動向資料
```

**活用方法例：**
```
「次の評価面談でアピールすべき実績を整理して」
「市場価値を上げるために今年やるべきことは？」
「AIスキルを本業でどう活かすか提案して」
「転職すべきか・今の会社に残るべきか判断材料を整理して」
```

---

### 02 Business
**定義：** 自分がオーナーとして構築する事業の意思決定

**活用目的：**
AIコンサル・個人事業の戦略設計、婚活業界への注力方針、
新規事業の優先順位判断を行う

**ソース例：**
```
・事業計画・方針メモ
・婚活業界のビジネストレンド・市場情報
・競合・参入機会の調査資料
・収益モデルのアイデアメモ
・AI実装ロードマップ（実績追記版）
```

**活用方法例：**
```
「今年注力すべき事業領域はどこか」
「婚活業界でAIを使ったビジネス機会を提案して」
「AIコンサルとして差別化できるポジションは？」
「事業の優先順位を現状ベースで整理して」
```

**拡張トリガー：**
```
・事業が複数になる
・業務領域が増える（営業/マーケ/開発）
・情報量が増えすぎて検索性が落ちる
→ Strategy / Marketing / Sales / Product / Ops に分割
```

### 03 RINGBELL
**定義：** 業務委託として価値提供する副業の実行・改善

**活用目的：**
婚活コンサルの業務ナレッジを蓄積し、
返信品質・面談精度・顧客満足を最大化する

**ソース例：**
```
・よく使う返信テンプレート集
・顧客対応FAQ・NG事例
・面談の進め方・チェックリスト
・婚活業界のトレンド情報（定期追加）
・過去の成功事例メモ（匿名化）
```

**活用方法例：**
```
「この状況の会員への返信文を生成して」
「この会員のプロフィール、どんな提案が刺さるか」
「面談前の準備で見落としがちなポイントは？」
「最近うまくいかないパターンの共通点を分析して」
```

**Claude Codeとの連携：**
```
返信生成AI・面談準備AIのベースナレッジとして活用
ここのソース品質が自動化ツールの精度を決める
```

### 04 Asset
**定義：** 全所有物の購入・所有・削減の最適化

**活用目的：**
衝動買いを防ぎ、所有物の質を上げ、
ファッション・ギア・ガジェットの意思決定を一貫させる

**ソース例：**
```
・所有アイテムリスト（Fashion / Gear / Everyday）
・体型・カラー診断データ
・スタイル方針・購入ルール
・過去のNG事例・失敗購入記録
・使用シーン別コーディネートメモ
```

**活用方法例：**
```
「このアイテム買うべきか？5軸で判断して」
「今のワードローブの偏りと不足を分析して」
「○○シーン向けのコーディネートを提案して」
「使用頻度が低いアイテムの処分候補を出して」
「このギアは今の競技レベルに必要か判断して」
```

**Beautyとの住み分け：**
```
Asset  → 美容アイテムの購入・所有管理（化粧水・香水など）
Beauty → 美容ノウハウ・知識・ルーティン設計
```

**Cultureとの住み分け：**
```
Athlete → 競技・パフォーマンス・身体最適化
Culture → 運動を含まない文化的・創造的活動
```

### 07 Life
**定義：** 人生設計・ライフイベント・資産形成の意思決定

**活用目的：**
結婚・住宅・資産形成・キャリア転換など
人生の大きな意思決定を構造的に考える
（Finance含む）

**ソース例：**
```
・今後のライフイベント計画メモ
・住宅ローンの基礎知識資料（金融庁・FP資料）
・NISA・資産運用の参照資料
・収入・支出・資産状況（概算）
・結婚後の生活費シミュレーション
・人生の優先順位・価値観メモ
```

**活用方法例：**
```
「住宅ローン、変動か固定かの判断基準を整理して」
「共働き前提でのキャッシュフロー設計をして」
「今年中に決めるべき財務上の優先事項は？」
「今後5年のライフイベントと必要資金を整理して」
「賃貸継続vs購入、今の状況でどちらが合理的か」
```

**Relationshipとの住み分け：**
```
Life         → お金・住宅・ライフイベントの設計
Relationship → 人との関係・コミュニケーションの設計
```

### 08 Relationship
**定義：** パートナー・家族・人間関係の意思決定

**活用目的：**
結婚を前提としたパートナーとの関係設計、
家族・友人・職場の人間関係における
コミュニケーションと意思決定を最適化する

**ソース例：**
```
・パートナーとの価値観・ルールのメモ
・家族との関係性メモ（匿名化）
・人間関係で繰り返す課題のログ
・コミュニケーションで学んだこと
・結婚準備に関する情報・チェックリスト
```

**活用方法例：**
```
「この状況でパートナーにどう伝えるべきか」
「結婚準備で決めるべき項目を整理して」
「繰り返している人間関係のパターンを分析して」
「価値観のすり合わせで重要な論点を出して」
「親との関係で改善できることはあるか」
```

### 09 Culture
**定義：** 文化的趣味・教養・創造活動の最適化

**活用目的：**
アート・音楽・楽器・読書・映画など
文化的活動の学習設計・深掘り・インスピレーション管理を行い、
人生の豊かさと創造性を最大化する

**ソース例：**
```
・始めた趣味の教材・書籍・動画書き起こし
・影響を受けたアーティスト・作品のメモ
・インスピレーションを受けた言葉・シーンの記録
・自分の創作ログ・練習記録
・学びたいジャンルの体系的な入門資料
```

**活用方法例：**
```
「ギターを始めるとして最初の3ヶ月の練習設計をして」
「この作品から学べる技術・表現を分析して」
「自分のライフスタイルに合うアートの方向性を提案して」
「読んだ本の核心を自分の言葉で再定義して」
「今の活動に創造性を取り込むアイデアを出して」
```

**Athleteとの住み分け：**
```
Athlete → 競技・パフォーマンス・身体の最適化
Culture → 創造・教養・感性の最大化
```

**拡張トリガー：**
```
特定の趣味が本格化して情報量が増えたら
→ 例：「Guitar」「Art」として独立ノートに分割
```

### 10 Journal
**定義：** 思考・感情・気づきの生ログ

**活用目的：**
日々の思考を音声で記録し、
月次で振り返ることで自己パターンを把握、
重要な洞察をObsidianに昇華させる

**ソース例：**
```
・Typelessで録音した音声ジャーナルのテキスト化
・日々のモヤモヤ・気づき・感情メモ
・意思決定の迷いのログ
・週次振り返りメモ
```

**活用方法例：**
```
「今月の自分の思考パターンを振り返って」
「繰り返し出てくるテーマや悩みは何か」
「今年の目標に対して行動が一致しているか」
「最近のストレスの共通点を分析して」
「意思決定の一貫性はあるか」
```

**Obsidianへの連携：**
```
音声ジャーナル（生ログ）→ Journal に蓄積
  ↓ 月次で分析
出てきた洞察・気づき
  ↓
Obsidian【09_Daily】にパーマネントノートとして保存
```

## 7. 拡張ルール

```
① ノートは増やしすぎない（上限12冊を目安）
② 増やす場合は「横に追加」する
③ 分割は必ず"情報過多"が起きてから
④ 迷ったらJournalに入れて後で分類
⑤ Cultureは趣味が本格化したら個別ノートに分割
```

### 例外ルール（Obsidian連携）

- 「原則NBLM内完結」のノートでも、**再利用価値が高い意思決定**はObsidianへ昇格する。  
- 「新規フォルダ検討」領域は、当面 `10_Knowledge` に集約し、**月10件以上**または**週次参照が常態化**した時点で専用フォルダを新設する。  
- 昇格時はノート末尾に「根拠ソース」「更新日時」を必ず残す。
