# 20_Input/04_LINE コンテキスト

## 役割

**個人**の LINE から逃がしたメモ・リンク・Keep 相当の断片の受け皿（週1コピペ等の **手動・準手動** を想定）。

## 業務との切り分け（必須）

- **副業・本業の公式 LINE（会員対応）** をここに自動投入しない。業務正本は各ドメイン（例: [[02_Business/01_RINGBELL/00_索引]]）のガバナンスに従う。
- **会話ログの一括自動流出**は行わない。対象は「自分用メモ・自分に送ったリンク」などに限定。

## 設計の正本

- [[Obsidian_RSSとLINE取り込み設計]]（`04_AI/03_学習/`）

## 運用ルール

- 貼り付けノートは先頭に **`date:`** と **`source: LINE`**（または固定1行）で出所を揃える。
- 資産化後は `05_Life/` 等へ移動（[[04_AI/01_運用/05_データ蓄積ガイド]]）。
- 自動化（メール橋・Make 等）は、**週1ルーティンで形式が固まってから**検討。

## Claude Code への指示方針

- 機微・会員情報が混ざっていないか確認してから処理すること。
- 不明な保存先は `00_Inbox/` に回し、人間に判断を仰ぐこと。
