# 20_Input/03_RSS コンテキスト

## 役割

RSS（購読フィード）から取り込んだ **記事メタ・抜粋・リンク** の受け皿。Clippings と同様、**資産化は別ステップ**（要約・ドメイン振り分けは `05_データ蓄積ガイド` を正とする）。

## 設計の正本

- [[Obsidian_RSSとLINE取り込み設計]]（`04_AI/03_学習/`）
- 購読URLの一覧（現行正本）: `20_Input/05_日次情報収集/_購読リスト.md`
- 補足: 日次の収集結果（`YYYY-MM-DD_collect.md`）は `20_Input/05_日次情報収集/` に保存する

## 運用ルール

- **フィードは少数**に保ち、週次で見直す（ノイズ抑制）。
- **全文スクレイプ**は各サイトの利用規約・robots・取得間隔に注意。RSS に本文が無い場合は **リンク＋要約** に留めるのが安全。
- 自動化バックログとの整合: [[04_AI/01_運用/06_自動化バックログ]]

## Claude Code への指示方針

- 取り込み済みノートを処理するときは、洞察・保存先ドメインを判断して転記すること。
- 出典 URL と取得日を残すこと。
