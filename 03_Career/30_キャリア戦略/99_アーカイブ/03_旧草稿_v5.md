# Part 1：現状認識（What is happening?）

## 1-1. 三つの断絶

AI時代の変化は「量的な延長」ではなく「質的な断絶」だ。三つの断絶を理解することが全ての出発点になる。

### 断絶①：AIの自律化（コパイロット → エージェント）

**2024年まで**：AIは「指示して動かすツール」だった。人間が「How（どうやるか）」を指示し続けることが前提。

**2026年現在**：AIエージェントは「目標を与えれば自律的に完結させる実行者」になった。計画・実行・エラー修正・品質チェックまでを人間の介入なしでループする。

```
旧モデル：人間が指示 → AIが実行 → 人間が確認 → 人間が次の指示
新モデル：人間が目標設定 → AIが計画・実行・自己修正 → 人間が最終判断
```

### 断絶②：競争軸の移動（能力 → 運用品質）

「どのモデルが賢いか」から「企業導入の障壁を誰が潰すか」へ。長時間実行・権限制御・監査可能性・メモリ保持——PoCが本番に上がれない理由のほとんどがここで詰まる。地味な部品が市場を制する。

### 断絶③：「ずっと考えられること」の希少化

> 「AIとの壁打ちが無限にできる時代だからこそ、本当に好きなテーマをずっと考えられることが唯一の優位性になる」（LayerX 福島良典）

情報格差・開発コスト・専門知識の壁がすべて消えた今、残る差異は「熱中できるテーマへの持久力」だけだ。

## 1-3. 「死滅回遊」としてのAI時代

Goodpatch土屋尚史氏の比喩。昨日まで普通の人間だった者に突然「術式（固有能力）」が与えられ、歴戦の術師と同じフィールドに放り込まれる。

**術式の強制付与**：Claude Codeを使えば、非エンジニアでもアプリが作れる  
**覚醒しただけでは勝てない**：「動くもの」と「人が使いたいもの」の間には分厚い壁がある  
**ゲームの真の目的が違う**：プラットフォーマーが見ているのは「人間がAIをどう使うかのデータ」だ  
**ルールが毎週変わる**：先月のベストプラクティスが今月の時代遅れ

**ただし決定的な違いがある**：これはゼロサムゲームではない。失敗コストがゼロに近い。100個作って1個当たればいい。死滅回遊のプレイヤーが聞いたら泣いて羨む環境だ。

**出木場の主張**：
- 「AI is not a Job Killer。AIはJobを殺すのではなく、Jobをリデザインする」
- 現時点でAIが雇用に大規模な影響を与えたデータはない

- 最も変わりやすいのはソフトウェアエンジニア。次は法律・経理だが、今のところ米国では採用増

```

徐々に・タスクから変化   ← 出木場が肯定しているのはここ

```

**行動への含意**：パニックになる必要はない。ただし「5年後に向けた投資」を今始めないのは危険だ。

# Part 2：構造変化（Why is this happening?）

## 2-1. 市場構造：SaaSの死と生

ライスキャピタルVC・福山太郎氏の分析（170社以上に投資、うち7社がユニコーン）：

**「SaaSの死」とは何か**  
SaaSそのものがなくなるのではない。**成長率が鈍化し、投資家の注目がAIエージェントに移る**ことを指す。

**消えていくSaaS vs 残るSaaS**

| 消えていくSaaS | 残るSaaS |
|---|---|
| 表面的なワークフロー・レポート作成 | システムオブレコード（基盤システム） |
| 「Excelの代替」として機能していたもの | 顧客データ・人事データ・支出データの管理 |
| 開発コストが下がれば内製されるもの | エラーが許されない・セキュリティが重要なもの |
| ライセンス課金だけが強みのもの | ドメイン知識×AI活用の深い実装 |

**最大の構造変化：課金モデルの転換**  
旧：ライセンス課金（1ユーザーあたり月額〇〇円）  
新：成果報酬・アウトカム課金（どんなアクションに対していくら）

AIエージェントが普及すると「5人の担当者が1万AIエージェントを使う」状況になる。そこに「人数ベース課金」は機能しない。

**SaaS×AIの相性がいい日本**  
日本の特殊性：労働人口の減少が急速で、フィジカルな現場（物流・医療・建設）のAIエージェント化が急務。グローバルプレイヤーより日本のローカルプレイヤーが有利な分野が多い。

---

## 2-2. 組織の再編：ピラミッドから砂時計へ

AIエージェントは中間管理職の典型的業務（情報集約・報告書作成・進捗管理）を最も得意とする。

```
【従来】ピラミッド型            【これから】砂時計型
      ▲                              ▲
     ▲▲▲  経営層                   ▲▲▲  経営層（天才×AI）
    ▲▲▲▲▲ 管理職（厚）           ▲ ▲  AIエージェント群（薄）
   ▲▲▲▲▲▲▲ 現場               ▲▲▲▲▲ 現場（少数精鋭）
```

**ワンピース型組織の実現**（KAIZEN Platform 須藤憲司）  
かつて漫画の中だけの話だった「少数の仲間でとんでもない大組織を相手にする」が現実になる。今まで100人必要だった仕事を、天才×AIで10人でできる時代。

ただし「人的資本の重要性が下がる」一方で、**天才の重要性が劇的に上がる**。天才が設計した業務をAIがひたすら再現する——これが次世代企業の構造だ。

---

## 2-3. タスク分解という競争軸

KAIZEN Platform 須藤氏の実践的フレームワーク：

**業務を最小タスクに分解すると、AIで担えるものが見えてくる**

例（営業業務の分解）：
```
① 業界リストアップ       → AIが担える
② 企業リストアップ       → AIが担える  
③ コンタクト先収集       → AIが担える
④ 企業現状・課題調査     → AIが担える
⑤ 提案内容をメール作成   → AIが担える
⑥ メール送信・返信対応   → AIが担える
⑦ アポイント調整        → AIが担える
⑧ ニーズヒアリング       → 人間がやるべき
⑨ 提案・プレゼン        → 人間がやるべき（ただしAI支援）
⑩ 交渉・クロージング     → 人間がやるべき
```

**重要な発見**：①〜⑦は自動化できる。しかし⑧〜⑩は「言語化できない悩み」への対応が必要で、人間が担うべき。AIは「受動的なオーダー実行」は得意だが、「能動的な示唆・交渉・コンサルティング」はまだ人間の領域だ。

---

## 2-4. AIエージェントは「労働力の代替」

福山太郎氏の本質的な整理：

**SaaSとAIエージェントの根本的違い**：
- SaaS＝**Excelの代替**（情報を整理・可視化するツール）
- AIエージェント＝**労働力の代替**（実際に仕事をする存在）

これが意味すること：今まで「人を雇う」か「システムを導入する」かだった選択が、「人を雇う」か「AIエージェントを使う」かになる。コスト構造が根本から変わる。

**ブルーカラーにこそチャンスがある**  
ホワイトカラー（特に中間層）が最も代替リスクが高く、フィジカルな仕事・身体を使う仕事の方が代替が難しい。AIエージェント＋ロボティクスで「ブルーワーカーのAI武装」が起きる時代に、むしろブルーワーカーの賃金が上がる可能性がある。

# Part 3：個人戦略（What should I do?）

## 3-1. 「入口と出口」フレームワーク

梶谷健人氏（POSTS代表）の分析。AIが進化するほど「中間の作業」が代替され、人間の価値は両端に集約される。

```
┌─────────────────────────────────────────────────────────┐
│                      業務の全体像                          │
├──────────┬────────────────────────────┬─────────────────┤
│  入口     │         中間（AIが担う）        │     出口        │
├──────────┼────────────────────────────┼─────────────────┤
│ 問いを立てる│ 資料作成・コーディング          │ A：レビュー・   │
│ 課題設定   │ 調査・リサーチ                 │    品質管理     │
│ 起案・推進  │ レポーティング・初稿生成        │ B：人を動かす   │
│           │ 定型分析・情報集約              │    交渉・合意   │
│           │ 議事録・メール初稿              │    判断責任     │
└──────────┴────────────────────────────┴─────────────────┘
    ↑                     ↑                        ↑
 高まる                 消える                    高まる
```

**中間領域で生き残れるのはトップ0.01%のみ**。3年かけて習得した「片の技」は、AI時代に最も不利なポジションになる。

---

## 3-2. 三つのキャリアパス

### パス①：起業家・事業責任者型（入口＋出口B）

問いを立て、AIで実行し、人を動かす。「やりたいことが無限に出てくる」人に向く。

**強みの源泉**：やりたいことがあること自体が最大の資産（広木大地）。AIとの壁打ちが無限にできる今、構想力と熱中テーマを持つ人間に飛躍的な差がつく。

### パス②：専門性×ディレクター型（出口A）

深い専門知識を持ち、AIアウトプットの品質を管理・方向づけする。

**強みの源泉**：ドメイン知識という「鑑定眼」。入山章栄教授：「AIを使って生産性10倍以上になれているのは、経営学のドメイン知識を持ち、アウトプットの善し悪しを判断できるから」。

**理想形**：「自分の出身ドメイン×ディレクター」——デザイン出身のAIプロダクトディレクター、法律出身のAI契約管理ディレクター、など。

### パス③：中間領域トップ0.01%（超例外）

特定の中間スキルでAIが到達できない水準に達する。そのスキルに「狂えるほど熱中できる」人だけが選べるルート。

## 3-3. スキル掛け算の設計法（BS型キャリア）

エッグフォワード徳谷智史氏の「キャリア3.0」概念と、深津貴之×梶谷健人の対話から抽出。

### PL型 vs BS型キャリア

| PL型（短期収益志向） | BS型（長期資産形成志向） |
|---|---|
| 今の年収が最大になる場所を選ぶ | 一時的に年収を下げてでも成長機会を掴む |
| 快適な環境を優先する | 今まで通用しない領域に敢えて踏み込む |
| 今できることを深掘りする | 将来の選択肢を広げる投資をする |
| 短期に最適化 | 障害年収の最大化を目指す |

**重要な洞察**：キャリア1.0時代は会社側がBS型投資（年功序列・終身雇用）を保証してくれた。キャリア3.0時代は**自分でBS型投資戦略を設計する**しかない。

### スキル掛け算の設計ステップ

**ステップ1：最初の2枚を市場原理で選ぶ**

人材価値＝希少性×市場性×再現性（徳谷智史）

```
100人に1人のスキルA × 100人に1人のスキルB = 1万人に1人の人材
```

良い組み合わせの条件：
- 同じ人が持つことが珍しい（例：法律知識×AIエンジニアリング）
- 一方が他方の価値を10倍にする（例：特定業界の深い知識×AI活用）
- 転職せずとも「場所を変える」だけで希少性が生まれる（例：IT系デザイン×製造業）

**ステップ2：フィニッシュパターンを設計する**

自分の「超必殺技（フィニッシュ）」を決め、そこに持ち込むための「浮かし技（サブスキル）」を取る。

例（営業が強みの場合）のフィニッシュ：クロージング  
→ サブスキル：その場でプロトタイプを作れる能力 / 相手の課題を即座に構造化する力

**ステップ3：3枚目以降は情熱で選ぶ**

35歳までに一度「ジャンプ」せよ——給料を下げてでも全く違う文脈の何かを始めること（PIVOT小野武彦）。デグレードではなくジャンプ。ジャンプしたら一時的に下がるのは当然だ。

## 3-4. I型→T型→H型→HH型という進化ロードマップ

エッグフォワード徳谷智史氏のフレームワーク。

```
I型    ：1つの深い専門性
T型    ：1つの深い専門性 + 周辺の幅広い知識
H型    ：2つの深い専門性 + 橋渡しの能力
HH型   ：3つ以上の深い専門性 + 複数のブリッジ（＝ブリッジ型）
```

**年代別の理想的な進化**：

| 年代 | 目標 | 具体的アクション |
|---|---|---|
| 20代 | I型→T型 | まず1本柱を徹底して深掘り。周辺も広げ始める |
| 30代前半 | T型→H型 | 2本目の武器を本気で作りに行く |
| 30代後半 | H型固め | H型が安定してきたら、もう1本を意識し始める |
| 40代以降 | H型→HH型 | 複数の柱が互いに価値を高め合う状態を作る |

**転職市場での判断基準**：車内・社外から固有名詞で声がかかるレベルが「本物の深さ」。「誰でもいいけど」ではなく「この仕事ならこの人」と言われる領域を持つこと。

**「プライドの鎧」に気をつけよ**：大企業の肩書きや快適な環境への慣れが、市場価値向上の機会を気づかずに阻む（徳谷）。定期的に社外の人間と自分の市場価値を確認する習慣が必要だ。

## 3-5. 自己配置力という新しい知性

LayerX Chief of Staff・櫻井氏の概念。環境や状況に応じて自分の立ち位置を変えられる力。

**三つの構成要素**：
- 自己位置づけ認知：今の自分がどのポジションにいるかを把握
- 盤面認知：組織・事業・イシューを俯瞰し、構造とボトルネックを理解
- 自己操作能力：「今、自分は出るべきか退くべきか」を判断して動く

# Part 4：必要な能力（What do I need?）

## 4-1. 優先度HIGH：絶対に磨くべき4つ

### ① 問いを立てる力（最重要・最希少）

AIは「解く」が得意だが「問う」は人間の領域。AIが進化するほど「何を解くべきか」を定義する能力の希少性が高まる。

鍛え方：毎日「なぜこれをやっているのか」を5回問い続ける。

### ② オーケストレーション能力（AIディレクション）

深津貴之氏の核心：「AIを使えない人の多くは実はディレクション能力がない自分を告白している。与えられた条件でツールが最も輝く環境を作ることがディレクションの本質だ」。

**4つの構成要素**：予見 → 指示の分解力 → 各プレイヤーの強みの把握 → 連携の設計

### ③ 人間力と関係資本

信頼は人間同士のやり取りの累積。複数の経営者が「スキルより、一緒に働いて楽しい人を採りたい」と言い始めている。これは感情論ではなく、ホワイトカラーの専門性がAIに代替されるほど「人間としての魅力」の相対的価値が上がるという合理的判断だ。

### ④ 変化を楽しめる心（最終的なアドバンテージ）

出木場社長の言葉：「30年前、父親から大きい銀行や大企業に行けば安泰と言われた。今の親はAI時代に生きられるよう教育しろと言っている。両方同じ間違いをしている。大事なのは変化が起きた時に『うわ、変わらなきゃ』と思うか、『新しい変化だ、楽しそう』と思えるか——その一点だけだ」。

変化を楽しめる人が幸せになる。それだけが、長期的に確かなことだ。

---

## 4-2. 優先度MID：あると差がつく3つ

### ⑤ 批判的評価力・審美眼

「良いものと悪いものの違いを言語化できる力」。昔は「頭でっかちの評論家」と馬鹿にされていたスキルが、AI時代の最重要スキルになった。

### ⑥ ドメイン深掘り

AIがあらゆる専門知識をオンデマンドで提供できる今、「AIと組み合わせることで誰も届けられない価値」を持つドメインエキスパートの希少性が上がる。

### ⑦ チェンジマネジメント力

カスタマーサクセス分野から見えてくる普遍的なスキル（山田ひさのり氏）：**組織が変わるロジックを理解し、小さな成功体験を積み上げて変革を実現できる力**。個人の変革と組織の変革の2段階を意識し、それぞれにアプローチできること。BtoB・社内変革・AI導入のどれにも効く。

---

## 4-3. 優先度LOW：AIに委ねていいもの

情報収集・調査・サマリー、コード初稿・ドキュメント初稿、定型分析・レポーティング、議事録・メール初稿、スケジュール管理・調整、定型的な翻訳、カスタマーサポートの定型回答（最大80%はAI対応可能・アディッシュ実証）

**心構え**：これらを委ねることで生まれた時間を、優先度HIGH/MIDに全振りすること。「楽になった」で終わると競争力は上がらない。

---

# Part 5：組織変革（How to transform organizations?）

## 5-1. 1%の設計者と99%の実行者

**「全員研修」が失敗する理由**：100人に研修→3ヶ月後に誰も使っていない→300万円のドブ捨て。

正しい構造：1%の設計者が「何をAIに、何を人間に」という制度を設計し、99%がそれを実行する。

**1%の設計者の条件**：
- 業務全体の横断的理解（複数部門経験）
- AIの可能性と限界の両方を実践で知る
- コスト感覚（「AIが勝つ部分」を数字で判断できる）
- 即断力（合意形成に逃げない）

**262の法則**：どの組織でも、上位2割は自発的に動く。下位2割は何をしても動かない。中間6割は空気で動く。**中間6割が積極層に転換する瞬間**が「社内PMF」だ。

---

## 5-2. モメンタムの4フェーズ

メルカリ AIストラテジー・ハヤカワ五味氏の実践から抽出。

**Phase 1：盛り下がりを先に潰す**  
「AIは万能」という過剰期待を先に下げる。苦手なことを正直に伝え、「ガッカリ」がモメンタムを殺す前に手を打つ。

**Phase 2：小さな火を守る**  
前向きな1〜2割が動ける環境を先に整備する。「使っていいツール」「確認不要の範囲」を明確化。

**Phase 3：社内PMFを狙う**  
AI活用度を評価に組み込む。経営陣自身の「最初の不安と乗り越え」を語らせる。成功者をMVPにして可視化する。

**Phase 4：経営のレールを引く**  
「AIを使わないと損、使えば評価される」制度を作る。LayerX：「トップが権限委譲という言葉に逃げないことが最も大事。誰よりもAIを触ってコミットした上で、一部を委譲せよ」。

---

## 5-3. ハイブリッドワークフロー設計

カスタマーサクセス・山田ひさのり氏の実践知：

**AIと人間のハイブリッドモデルを早期に設計せよ**

誤った移行パターン：「20%が自動化できてない → 使えない → 元に戻す」  
正しい移行パターン：「20%が自動化できた → 比率を高める目標を設定 → 毎四半期で改善」

**具体例（カスタマーサポート）**：
- 現状：チケット対応の30〜50%をAIが処理
- 目標：最大90%をAIが処理（残り10%は人間が必ず担う）
- 設計：AIが諦めたものだけ人間に来るワークフローを整備
- 効果：コスト削減＋均一性の確保＋ベストプラクティスの発見

**SaaS市場への含意**：カスタマーサポート業務のAI化が最大80%（アディッシュ実証）まで可能。これが人的資本を「ルーティン対応」から「チェンジマネジメント」へ移す機会となる。

# Part 6：隠れたリスク（What could go wrong?）

## 6-1. 意思決定疲労（AI Brain Fry）

HBR（2026年2月）：「AIは仕事を減らさない。むしろ仕事を強化（Intensify）する」。AIが高速化すると、人間はより多くのプロジェクトを並走させられ、「終わりのない意思決定ループ」に陥る。

**対策**：「AIを止める時間」を意図的にカレンダーに確保。全判断を自分が引き受けない仕組みを作る。

---

## 6-2. AI迷子の増加

オープンワーク大沢氏：2025年から求人検索でAIというキーワードが急増。「何かしなければ」という漠然とした不安が満ちている。AI迷子を防ぐ鍵：**AIを「最初の強み」にしない**。自分のコアスキルを別に持った上でAIを乗数として使う。

---

## 6-3. 「殻」が「檻」になる瞬間

ひろゆき×坂井風太×成田修造の対談から：

大企業は最初「殻」として機能する——履歴書が強くなり、転職もしやすく、給料も安定する。しかしある時点で「檻」に変わる——快適すぎて出られなくなり、市場価値が車内評価としか繋がらなくなる。

**坂井風太の経験談**：「これ、あと3年いたら確実に檻になると気づいた。だからまず住宅ローンだけ組んでからやめた」。

**分岐点の見極め方**：「この会社を出ても同じことができるか？」という問いに自信を持ってYesと言えない状態が続いているなら、すでに殻が檻になりかけているサインだ。

**「どこでも働ける」感覚を常に持つ**：実際に転職しなくていい。転職活動（情報収集・面接の感触）だけ定期的に行い、自分の市場価値を確認し続けること（PIVOT小野武彦）。

---

# Part 7：日本市場の特殊性（Japan context）

## 7-1. JTCという選択肢の再評価

PIVOT・小野武彦氏の逆説的分析：AIによる雇用不安が高まるほど、雇用が安定しているJTCの相対的価値が上がる。

しかし**選抜基準が変わる**：「間違えないインデックス型エリート」から「野生的なリーダー」へ。手を動かして実際にものを作れる、決断から逃げない、責任を取れる——これが新しいJTCエリートの条件だ。40代での大企業トップが当たり前になる時代が来る。

---

## 7-2. 大企業は「殻」か「檻」か——分岐の見極め方

| 「殻」として機能している状態 | 「檻」になっている状態 |
|---|---|
| 新しいスキルが積み上がっている | 既存スキルの消費のみ |
| 社外でも通用する実感がある | 社内の人間関係だけが頼り |
| 「やめる理由がないから続けている」 | 「やめる勇気がないから続けている」 |
| 転職市場での価値が把握できている | 自分の市場価値が不明 |
| 意思決定の機会が増えている | 承認の待ちが業務の大半 |

---

## 7-3. 転職市場の変化と2つの失敗パターン

**大企業転職特需のラストイヤー**（オープンワーク大沢）：DX系・エンジニア系の大手求人は埋まりつつある。今後残る求人はAI/GX関連、第2新卒特定職種、同一業界内移籍が中心。

**転職の2つの失敗**（PIVOT小野武彦）：

①裏切り転職：辞めた会社に不義理をする。業界内のレピュテーションは長く残る。年収が上がれば上がるほど、採用企業はリファレンスに時間をかける。

②自分で決めなかった転職：親・配偶者・同僚の圧力で決めた転職。矢印が自分に向かわないため、うまくいかない時に引きずり続ける。「好きでもない人と結婚したようなもの」。

---

## 7-4. 採用基準の変化

| 観点 | 旧（〜2024） | 新（2025〜） |
|---|---|---|
| スキル | 特定技術の深い知識 | AI活用できる前提で、掛け算できるスキル |
| 経験 | 同職種での実績 | ドメイン知識×変化への対応力 |
| 人物 | 会社のルールに従える | 一緒に働いて楽しい、「いいやつ」 |
| AIリテラシー | あると加点 | ない人は選考外になりつつある |
| コミット | 会社へのロイヤルティ | ミッション・ビジョンへの共感 |

# Part 8：職種別影響度（Industry & Job Analysis）

## 8-1. 職種別AI代替リスクマップ

リクルートHD出木場・KAIZEN須藤・マッキンゼーデータの統合：

**高リスク（5〜10年で大きく変化）**

| 職種 | 変化の内容 |
|---|---|
| ソフトウェアエンジニア（ジュニア） | 採用はすでに米国で40%減。コーディング業務の大半がAI代替へ |
| カスタマーサポート | 定型対応の最大80〜90%が自動化可能 |
| 事務・バックオフィス | 米国大手企業でバックオフィスの新規採用を停止始める |
| 法律・会計（定型業務） | 契約書レビュー・経費処理の大部分が代替可能 |
| アナリスト・レポート作成 | データ集計・初稿作成はほぼ代替可能 |

**低リスク（当面は人間が担う）**

| 職種 | 理由 |
|---|---|
| フィジカルな仕事（建設・介護・配管等） | 身体・空間判断が必要 |
| 高度な意思決定・責任を取る役割 | AIは責任を取れない |
| 深い人間関係に依存するセールス | 長期的な信頼構築はAIに代替困難 |
| 戦略コンサルティング（最上位） | 曖昧な問いから示唆を出す力 |
| ヘルスケア（看護・診察等） | 資格・身体接触・感情対応 |

---

## 8-2. SaaSとカスタマーサクセスの未来

**CS（カスタマーサクセス）の進化**（山田ひさのり氏）：

CSはかつて「継続率を上げる部門」だった。今は「アップセル・クロスセルを牽引するプロフィットセンター」になりつつある。背景：大手企業がSaaSに参入し、アカウントプランの重要性が高まった。

**AIがCSを変える3つの軸**：

①定型業務の自動化：オンボーディングの定型部分は1〜2年でほぼAI対応可能。シンプルなプロダクトのオンボーディングはAIアバターが担う時代が来る。

②非構造化データの構造化：ヘルスコアの7割は構造化データ（プロダクト利用率等）で説明できるが、残り3割は「関係の深さ・意思決定者との繋がり」という定性情報。AIはミーティング記録・メール履歴・会話トーンを解析し、これを構造化する。

③高度な業務への人的資源の集中：自動化で生まれた余裕を「チェンジマネジメント」「ネゴシエーション」「エクスパンション設計」に集中させる。

**CSで残るスキル**：
- チェンジマネジメント（組織変革を促進する力）
- ネゴシエーション・セールスエッセンス
- 中長期のアカウント設計（どの指標達成でエクスパンション提案をするかの握り）

---

## 8-3. 無形商材×法人営業という選択肢

サラタメ氏の分析：AI時代に最も汎用性が高く、今から未経験でも入れる職種。

**AI時代に残る4つの要素**と法人営業の関係：

| 要素 | 内容 | 法人営業との関係 |
|---|---|---|
| イレギュラーな運動 | AIには謎とも言える予測不能な行動 | 手書きの手紙、ゴルフ接待、突発的な対応 |
| 人間関係の積み上げ | 長期的な信用構築 | 法人との長期関係（個人より安定） |
| グレーな交渉 | 白黒つけすぎない人間的判断 | 意図的に折れて次の伏線を作る |
| ずるいポジション取り | 身軽な移動で成長市場に乗る | 無形商材は業界横断移動が容易 |

**無形商材が有形商材より優れている理由**：業界を移るのが容易。IT→人材→コンサルと移動できる。転職市場で「無形商材法人営業経験」は最も幅広い業界への入り口になる。

**注意点**：AIは「あなたに個別対応できます」という強みを持つが、「長年の信頼」「曖昧な合意」「感情的な結びつき」は人間のアドバンテージとして残る。

## 9-3. 市場価値の3軸評価法

エッグフォワード徳谷氏のフレームワーク。自分の市場価値を客観的に把握するための問い。

**軸①：希少性（どれだけ珍しいスキルの組み合わせを持っているか）**

問い：「この能力の組み合わせを持つ人は、日本に何人いるか？」  
判断基準：固有名詞で声がかかるか。「誰でもいい」ではなく「あなたに頼みたい」と言われる領域があるか。

**軸②：市場性（その能力が社会から求められているか）**

問い：「5年後も同じ需要があるか？AIに代替されないか？」  
確認方法：転職サイトで自分の経験の求人数を調べる。成長市場か縮小市場か。

**軸③：再現性（他の組織でも同じ価値を発揮できるか）**

問い：「この組織を離れても、同じことができるか？」  
確認方法：社外でのプロジェクト経験、副業、転職経験があるか。車内ルールや人脈に依存していないか。

## まとめ：問うべき唯一の問い

AIが普及するほど、最後に残る競争優位はシンプルだ。

**「仕事を奪われる」という議論をしてる時間があるなら——**  
まず今日の業務をAIに渡せる形で分解しろ。中間業務を委ねた時間で、問いを立てる練習をしろ。そして35歳までに、一度ジャンプしろ。

**変化を恐れるか、楽しめるか。それだけが最終的に効いてくる**（リクルートHD出木場）。

> **「自分は何のために、誰のために、ずっと考え続けられるか」**

この問いへの答えを持っている人間が、この時代を生き抜く。
