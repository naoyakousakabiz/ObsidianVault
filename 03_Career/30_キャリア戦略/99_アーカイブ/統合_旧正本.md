## Executive Summary（結論の仮説）

1. **技術軸**：AIはコパイロットからエージェントへ移行しつつあり、企業導入のボトルネックは「賢さ」より**運用・権限・監査・長時間タスクの制御**側に移っている（一方で、実装度は企業・職種で不均一）。
2. **個人軸**：価値の相当部分が**問いの設定（入口）**と**判断・合意・責任（出口）**に寄るという仮説は強いが、「中間が全滅」は**過剰単純化のリスク**が大きい。
3. **マクロ軸**：**雇用の急崩壊**を主張する説と、**ジョブのリデザイン・遅い変化**を主張する説は、データの定義と時間軸をそろえないと**疑似対立**になりやすい。
4. **日本軸**：雇用慣行・購買慣行・言語市場により、**海外ナラティブの直輸入**は誤予測を生みやすい。

## Part 0：セキュリティ市況とキャリア含意（Mythos）

**概要（整理メモ）**  
Anthropic が **脆弱性発見・悪用可能性を含む能力**を持つフロンティア系モデルとして **Claude Mythos Preview** を公表した、という文脈がある。一般向け無制限公開ではなく、**限定パートナー・防御目的**などの枠組みで議論されている（**詳細は公式を正**）。

**公式・一次に近い参照（リンクは随時更新）**

- Anthropic（紹介ページの例）: https://red.anthropic.com/2026/mythos-preview/
- リスク・説明の文脈: https://www.anthropic.com/ （サイト内検索）

**SNS転載メモ（未検証のまま保持しない用途）**  
株価・下落率・因果は**チャート・IR・報道本文で照合**すること。セキュリティ業界全体への打撃は**単線因果ではない**（防御投資・規制・インシデント需要など相反要因）。

**キャリア含意（仮説）**

- 「AI が脆弱性リサーチ等をどこまで代替・増幅するか」は、**セキュリティベンダ・クラウド・エンタープライズ IT** の議論と市場のボラティリティに接続されうる。
- 逆に **ガバナンス・監査・インシデント対応・安全な導入設計**（組織的な AI 利用統制を含む）の人材価値は相対的に上がりうる（**確定ではない**）。

**プロ視点の反論・保留**

- 「攻撃能力が高いモデル＝セキュリティ人材の価値下落」は短絡的。実務は**責任・証跡・運用・人間の意思決定**が中心で、モデル性能と**需給曲線**は別問題。
- 規制・顧客要求が強まるほど、**人員は減っても“詰めの”は残る**というパターン（過去の自動化と同型）がありうる。

## Part 1：現状認識 — 三つの断絶と、その批判

### 1-1. 断絶①：コパイロット → エージェント

**主張（要約）**  
指示単位の支援から、目標提示に対する自律的ループへ。

**反論・限定条件（プロ視点）**

- 「介入なしで完結」は**ドメイン・権限・失敗コスト**が低い領域に偏りがち。医療・金融・契約・顧客対応などでは**人間ゲートが制度要件**になりやすい。
- エージェントほど**失敗の見えにくさ（静かな品質劣化）**が問題になり、結果としてレビュー人員が増えるケースがある（後述の HBR 系議論と接続）。

### 1-2. 断絶②：競争軸が「能力」から「運用品質」へ

**主張（要約）**  
PoC が本番に上がらない理由の多くが、長時間実行・権限制御・監査ログ・メモリ設計など**地味な運用要件**に詰まる。

**反論・限定条件**

- 運用品質は**勝ち筋**の一方で、コストセンター化しやすく、評価されにくい職能でもある（「泥臭いがKPIに出ない」）。
- 「モデル企業がすべて食う」論は、**規制産業・垂直ドメイン・現場最適化**で繰り返し修正されてきた歴史がある（汎用基盤と現場実装のギャップ）。

### 1-3. 断絶③：「ずっと考えられるテーマ」の希少化

**主張（要約）**  
情報・実装・専門知識へのアクセスコストが下がるほど、差は**持続的注意・動機づけ・価値判断**側に移る、というナラティブ。

**反論・限定条件**

- 「熱中」は**健康・家庭・経済不安**とトレードオフになりやすく、構造問題を個人の努力に回帰させすぎるリスク（**モラリズム的キャリア論**への批判）。
- 組織側の設計（評価・裁量・心理的安全性）が悪いと、個人の「熱中」は**搾取対象**に転じうる。

### 1-4. 比喩（死滅回遊）の功罪

**効用（メモ２・３の要旨）**  
Goodpatch 土屋尚史氏の比喩として紹介されることが多い。呪術廻戦の「死滅回遊」では、昨日まで普通の人間だった者に突然「術式（固有能力）」が与えられ、歴戦の術師と同じフィールドに放り込まれる——AI 時代も、**術式の強制付与（例: Claude Code で非エンジニアもアプリが作れる）**、**覚醒しただけでは勝てない（動くものと使われるものの差）**、**ルールが毎週変わる**、といった対応づけが語られる。

**決定的な違い（メモ側の主張）**  
原作の死滅回遊とは違い、**ゼロサムではない**、**失敗コストが相対的に低い**、ニッチ課題が個人でも解けうる、という楽観もセットで語られる。

**プロ視点の反論（徹底）**

- 比喩は**ゼロサム・生存競争**を想起させがちだが、多くの市場は**需要拡大・新しい仕事の創出**を伴う。
- 「術式が与えられた」は**アクセス格差（予算・職位・情報システム政策）**を隠しやすい。
- 「失敗コストゼロ」は**顧客・契約・法務・レピュテーション**がある職務では誤り。スタートアップ文脈と JTC 文脈を混同しない。
- 少年漫画比喩は**ジェンダー・年齢**で刺さり方が分かれ、面接では使いどころを選ぶべき。

## Part 3：マクロ対立軸 — 「雇用破壊」対「リデザイン」対「遅いが確実」

### 3-1. 悲観ナラティブ（代表論点）

- ホワイトカラーの中間業務の自動化・エージェント化  
- ジュニア層・定型業務からの需要シフト  
- 「量産型ホワイトカラー」の価値下落説

### 3-2. リクルート HD 出木場氏系の慎重ナラティブ（メモ３で整理されていた軸）

- **Job killer ではなく job redesign**  
- 雇用統計への影響は**現時点では限定的**という主張もありうる  
- 変化は**タスク単位で徐々に**、ただし時間軸は長い

### 3-3. 統合的読み（対立を解くための枠）

```
急激な全失業（単純化されがち）  ← データで一括否定/肯定はしにくい
タスクの再配分（中間の薄化）  ← 職種により不均一に起きうる
企業内の政治・制度の惰性        ← 技術可能性より遅れやすい
```

**プロ視点の第三の批判**

- **「両方あって矛盾しない」**を装う統合が、かえって**何も予測していない**ことになる危険。  
- 個人戦略としては、**マクロの正解当て**より「自分の職務のタスク分解＋顧客価値＋証跡」の方が堅い（後述 Part 8 と連動）。

## Part 4：市場構造 — SaaS・エージェント・「労働力代替」

### 4-1. 「SaaS の死」とは何か（定義の整理）

**主張（要約）**  
SaaS カテゴリが消えるというより、**成長期待の座標**が「情報整理」から「実行・成果」側へ移る、という論点整理として有用。

**消えやすい / 残りやすい（メモ３の整理を統合）**

| 圧力がかかりやすい | 残存しやすい |
|:---|:---|
| 表面的ワークフロー・レポート作成の薄い価値 | システム・オブ・レコード、ミッションクリティカル領域 |
| 内製化しやすい汎用領域 | 規制・監査・セキュリティ要件が強い領域 |
| ライセンス課金のみが価値の中心 | ドメイン実装・運用込みの価値 |

**反論・限定**

- 「SaaS 死す」は**マーケ用語**になりやすい。実務は長期契約・データ移行・ワークフロー埋め込みで**粘る**。  
- **成果課金**への移行は、契約・法務・検収で**摩擦が大きく**、すぐには広がらない。

### 4-2. AI エージェント＝「労働力の代替」という主張への反論

**主張の強み**  
コスト構造の比較対象が「人」に近づくという発想転換は、採用・外注・BPO 設計に効く。

**反論**

- 現状は多くの場面で**「人＋AI」のハイブリッド**が費用対効果の最適解になりやすい。  
- **責任・説明・クレーム対応**は人間が残り、単純な人員削減にならない。  
- 「ブルーカラーの賃金上昇」など長期命題は**地域・制度・移民政策・ロボット価格**に依存し、断定は危険。

### 4-3. 競争軸：ハーネス設計（メモ２）とタスク分解（メモ３）

**ハーネス（運用ラッパ）**  
終了条件の外部化、視覚的検証、決定論ゲートなど。

**タスク分解（営業例など）**  
①〜⑦が自動化候補、⑧〜⑩が人間、という**示唆**は強いが、実際は**顧客ごとに⑧が前に崩れる**ことも多い。

**プロ視点の反論**

- 分解は**組織の政治**を無視しがち（誰が定義権を持つか）。  
- 「人間がやるべき」を**過大に神聖化**すると、単に**安い労働**に押し出す正当化にもなる。

## Part 5：組織 — ピラミッドと砂時計、エージェンティック・チーム

### 5-1. 砂時計化・中間管理職の二極化

**主張の核**  
情報集約・進捗管理・資料作成がエージェント適性が高い。

**図解（メモ２・３の ASCII を統合）**

```
【従来】ピラミッド型            【これから】砂時計型
      ▲                              ▲
     ▲▲▲  経営層                   ▲▲▲  経営層（天才×AI 等の語り）
    ▲▲▲▲▲ 管理職（厚）            ▲ ▲  AIエージェント群（薄い中間）
   ▲▲▲▲▲▲▲ 現場                 ▲▲▲▲▲ 現場（少数精鋭という語り）
```

**実例として語られがちなもの**  
大規模顧客対応の効率化、採用プロセスの設計変更など（**各社 IR・報道で要確認**）。

**反論**

- **レイオフの理由は複合**（需要・金利・M&A・株主還元）で、AI 単因にしないのがプロ。  
- 中間管理職の価値は「進捗管理」だけではなく**育成・倫理・心理的安全・跨部門調整**であり、そこは簡単に消えない。  
- 「少数精鋭」は**残業・燃え尽き・採用難**の外部化でもある。

### 5-2. エージェンティック・チーム（2〜5 人が多数エージェントを監督）

**反論**

- 監督コストと**責任の所在**が複雑化する。  
- セキュリティ・コンプライアンス上、**エージェントに渡していい権限**の設計がボトルネック。  
- 「少数天才」モデルは**多様性・後継育成**と衝突しやすい。

### 5-3. 1% の設計者と 99% の実行者

**主張の核**  
全員研修より、設計と環境（レール）が先。

**プロ視点の強い反論**

- **民主性・納得形成**を捨てると現場抵抗が増え、結局遅れる。  
- 「1%」は**単一障害点**。退職・燃え尽きで崩壊しうる。  
- 評価制度への組み込みは**ゲーミング**（見せための AI 利用）を生む。

### 5-4. モメンタム設計（期待値管理 → 小さな火 → 社内 PMF → レール）

**反論**

- 「社内 PMF」は**成功事例の後付けラベル**になりやすい。  
- Phase 設計は**文化依存**（心理的安全性が低い組織では逆効果）。

### 5-5. LayerX 型の実装例（トークン KPI 等）

**含意**  
トップコミットと予算設計の話として参考になる。

**反論**

- スタートアップ最適解が**大企業 JTC にそのまま転用**されない。  
- トークン KPI は**コスト最適化と逆行**しうる（無駄な推論を誘発するインセンティブ設計ミス）。

## Part 6：個人戦略 — 入口／出口・パス・スキル設計

### 6-1. 入口と出口（梶谷氏系の整理）

```
┌─────────────────────────────────────────────────────────┐
│                      業務の全体像                          │
├──────────┬────────────────────────────┬─────────────────┤
│  入口     │         中間（AIが担いやすい領域）   │     出口        │
├──────────┼────────────────────────────┼─────────────────┤
│ 問いを立てる│ 資料作成・コーディング          │ A：レビュー・   │
│ 課題設定   │ 調査・リサーチ                 │    品質管理     │
│ 起案・推進  │ レポーティング・初稿生成        │ B：人を動かす   │
│           │ 定型分析・情報集約              │    交渉・合意   │
│           │ 議事録・メール初稿              │    判断責任     │
└──────────┴────────────────────────────┴─────────────────┘
    ↑                     ↑                        ↑
 価値が寄りやすい        圧力がかかりやすい        価値が寄りやすい
```

**メモ原典の強い主張（要検証）**  
中間領域で生き残れるのはトップ 0.01% のみ、など。**職種によって中間が顧客価値の本体**であるケース（対面営業、導入コンサル、療養介護等）では当てはまらない。

**反論**

- 「中間は消える」は職種によっては**顧客接点が中間に宿る**（営業・CS）。  
- 入口も、上位者の**丸投げ問い**として表面化し、実質は出口の政治に飲まれることがある。

### 6-2. 三つのキャリアパス（統合）

1. **事業責任者型**：入口＋出口 B  
2. **専門×ディレクター型**：出口 A（鑑定眼）  
3. **中間の超上位（例外的）**：極めて狭い

**反論**

- パス分類は**自己啓発の棚**になりやすい。採用市場では**職歴・実績の言語化**に落とさないと意味が薄い。  
- 「AI は乗数」は正しいが、**乗数にする母数（ドメイン）が弱い**と差がつかない。

### 6-3. スキル掛け算・フィニッシュパターン（深津×梶谷系）

**実務コツ**  
最初の 2 枚は市場原理、3 枚目以降に情熱、など。

**反論**

- 「希少な掛け算」は**ニッチすぎて市場が小さい**こともある。  
- 35 歳ジャンプ等の年次ルールは**産業・職能で無効**なことが多い（批判的に使う）。

### 6-4. PL 型と BS 型（メモ３）

**使い方**  
短期最適と長期オプション価値のトレードオフを自問する道具。

**反論**

- 生活防衛・育児・介護がある人にとって PL 偏重は**必然**であり、道徳化しない。  
- BS 投資は**現職の剥奪リスク**（学んだが評価されない）もセットで検討する。

### 6-5. I 型 → T 型 → H 型 → HH 型（徳谷氏系）

**反論**

- 型の多さ自体が**不安産業**になりうる。実務では **1 本深い柱＋1 本橋**で十分な人が多い。  
- 「社外から固有名詞で呼ばれる」は理想だが、**市場の小ささ**で達成感が錯覚になることもある。

### 6-6. 自己配置力（櫻井氏系）

**反論**

- 自己配置は**権限がない組織では空文句**になりやすい。  
- 「盤面を読め」は**構造暴力の言い換え**になりうる点にも注意。

## Part 7：能力の優先度 — HIGH / MID / LOW と批判

### 7-1. HIGH（統合版）

1. **問いを立てる力**  
2. **オーケストレーション（分解・連携設計）**  
3. **人間力・信頼形成**（関係資本）  
4. **変化を学習対象として扱うメタ認知**（メモ３で強調されていた軸）

### 7-2. MID

- 批判的評価・審美眼  
- ドメイン深掘り  
- チェンジマネジメント（メモ３の CS 文脈を一般化）

### 7-3. LOW（AI に寄せる候補）

情報収集、初稿生成、定型レポーティング等。

**プロ視点の強い反論（「いいやつ採用」ナラティブへの）**

- 「スキルより人間力」は**採用バイアス・同質化・違法性リスク**（差別・ハラスメント見えない化）を生みうる。  
- 表向き「いいやつ」は、実態は**カルチャーフィット名の排除**であることがある。  
- 面接での対策は「人間力」ではなく**協働の証跡（苦渋の意思決定・越境・顧客成果）**で語る方が堅い。

## Part 8：リスク — 意思決定疲労、AI 迷子、「殻と檻」

### 8-1. AI Brain Fry（仕事の強化）

**主張**  
AI が速いほど、並行プロジェクトと承認が増え、疲労が増えるという注意。

**反論・補足**

- 組織設計（WIP 制限・責任分界）で**かなり緩和可能**。  
- 「AI が悪い」ではなく**マネジメントの失敗**として見るべきケースが多い。

### 8-2. AI 迷子

**主張**  
ニュース追いだけで実務が変わらない状態。

**反論**

- 逆に**過剰実験**もリスク（セキュリティ・著作権・顧客データ）。  
- 「何かしなければ」は**ベンダー煽り**とも共謀しうる。

### 8-3. 殻と檻（大企業アレゴリー）

**反論**

- 大企業は**キャッシュ・信用・リスク耐性**の「殻」として合理的な人も多い。  
- 転職肯定は**機会損失**とも言える。重要なのは**自らのオプション価値の計測頻度**。

## Part 9：日本市場 — JTC・転職・採用

### 9-1. JTC の再評価（相対的安定）

**反論**

- 安定は**賃金・スキル更新・配置の恣意性**とトレードオフになりやすい。  
- 「野生的リーダー」は**組織の言語**と衝突し、評価されないリスク。

### 9-2. 転職市場（求人のパターン変化説）

**反論**

- 求人票は**景気・金利・業界再編**の影響が大きい。  
- 「ラストイヤー」系は**印象的だが検証が難しい**ラベルとして扱う。

### 9-3. 採用基準の変化表（メモ２・３の統合）

**使い方**  
面接側の仮説として「こう聞かれうる」と準備する。

| 観点 | 旧（〜2024）として語られがち | 新（2025〜）として語られがち |
|:---|:---|:---|
| スキル | 特定技術の深い知識 | AI 活用が前提で、**掛け算**できるスキル |
| 経験 | 同職種での実績 | ドメイン知識×変化への対応力 |
| 人物 | 会社のルールに従える | 一緒に働いて楽しい人間力（「いいやつ」ナラティブ） |
| AI リテラシー | あると加点 | ないと減点・選考外、などと語られがち |
| コミット | 会社へのロイヤルティ | ミッション・ビジョンへの共感 |

**反論**

- 表は**ベンチャー寄り**に歪みやすい。JTC は**職能資格・年功・社内労働市場**が残る。  
- 「AI 使えて当然」は**定義が曖昧**（ChatGPT 触っただけ、など）。  
- 「いいやつ」重視は**多様性・法的リスク**（差別・同質化）と両立が難しい。面接対策は**成果と協働の証跡**へ寄せる。

## Part 10：職種別 — 影響度マップと反論（メモ３中心）

### 10-1. 職種別の影響マップ（メモ３の表を統合・**要照合**）

**高リスク（5〜10 年で大きく変化しうる、という語り）**

| 職種 | 変化の内容（メモ上の主張） |
|:---|:---|
| ソフトウェアエンジニア（ジュニア） | 採用は米国ですでに減少、コーディング業務の大半が AI 代替へ、など |
| カスタマーサポート | 定型対応の **80〜90%** が自動化可能、など |
| 事務・バックオフィス | 米国大手で新規採用停止、などの語り |
| 法律・会計（定型業務） | 契約レビュー・経費処理の大部分が代替可能、など |
| アナリスト・レポート作成 | データ集計・初稿作成はほぼ代替可能、など |

**低リスク（当面は人間が担いやすい、という語り）**

| 職種 | 理由（メモ上） |
|:---|:---|
| フィジカルな仕事（建設・介護・配管等） | 身体・空間判断が必要 |
| 高度な意思決定・責任を取る役割 | AI は責任を取れない |
| 深い人間関係に依存するセールス | 長期的な信頼構築は代替困難、など |
| 戦略コンサルティング（最上位） | 曖昧な問いから示唆を出す力、など |
| ヘルスケア（看護・診察等） | 資格・身体接触・感情対応 |

**反論（表全体に対して）**

- 「ジュニアが消える」は**教育コストと責任**の観点から、組織によっては逆に**ジュニアを増やす**選択もありうる。  
- CS の自動化は**顧客満足・解約・ブランド**とのトレードオフ。

### 10-2. CS の未来（山田ひさのり氏系の整理＋批判）

**主張の要点**  
定型の自動化で余力を、チェンジマネジメント・交渉・アカウント設計へ。

**反論**

- アップセル中心設計は**顧客利益と緊張**する。  
- 「AI がヘルススコア」は**プライバシー・説明責任**が重い。  
- CS は**セールス化圧力**とセットで読まないと、現場の実態がずれる。

### 10-3. 無形商材×法人営業（メモ３）

**主張の骨子（サラタメ氏系として紹介されていた整理）**  
AI 時代に汎用性が高く、未経験からも入りやすい職種として**無形商材×法人営業**が挙げられがち。

| 要素 | 内容 | 法人営業との関係（メモ上） |
|:---|:---|:---|
| イレギュラーな運動 | 予測不能な行動 | 手紙・接待・突発対応、などの語り |
| 人間関係の積み上げ | 長期的信用 | 法人との長期関係、など |
| グレーな交渉 | 白黒つけすぎない判断 | 折れて次の伏線、などの語り |
| ずるいポジション取り | 身軽な移動 | 無形商材は業界横断移動が容易、などの語り |

**反論**

- 接待・非定型は**規制・コンプライアンス**で縮小しうる。  
- 「無形商材は移動しやすい」は**営業ノルマと垂直専門性**の壁で簡単ではない。  
- AI は「個別最適のメッセージ」を量産しうるため、**「手書き＝希少」自体がコモディティ化**しうる。

## Part 12：行動計画 — 90 日・チェックリスト・市場価値 3 軸

- [ ] 「中間」に費やしている時間を計測する（体感で **50–70%** などと言われがちだが、**実測**を優先）  
- [ ] AI が既にできる業務に印をつける（**品質基準**も併記）

**Week 3–4：自己配置力・テーマの診断**

- [ ] （メモ３追補）希少性×市場性×再現性の 3 軸で自分を仮採点する

**Month 1 の成果物（目安）**

#### Month 2：実験フェーズ（小さく試す）

**Week 5–6：AI 活用の高速化実験**

- [ ] 浮いた時間を優先度 HIGH のスキル強化に再投資する

**Week 7–8：スキル掛け算・市場接続**

- [ ] 自分の「コアスキル（1 枚目）」を定義する  
- [ ] 「2 枚目のカード」を市場原理で選ぶ（需要×自分の接点）  
- [ ] 社内外で知見を **1 回** 発信する（社内 LT、ブログ、コミュニティ等）  
- [ ] （メモ３追補）エージェントと**転職目的でなくてよい**カジュアル面談をする  
- [ ] （メモ３追補）「殻にいるか／檻にいるか」を自己点検する

**Month 2 の成果物（目安）**  
スキル掛け算の設計図、AI を使った業務高速化の**Before/After**（数値がなくてもよいが、**観察メモ**は残す）

- [ ] 35 歳までに「ジャンプ」すべき機会があるか検討する（**年齢は目安**、産業で無効化されがち）

**Week 11–12：持続可能な仕組み**

- [ ] 意思決定疲労を防ぐ「**AI を止める時間**」をカレンダーに固定する

- [ ] メンターまたはピアグループを **1 つ** 見つける（なくてもオンラインコミュニティで可）  
- [ ] （メモ３追補）「変化を楽しむ」習慣を**小さく設計**する（ニュース断食デー等）

**Month 3 の成果物（目安）**  
3 年ロードマップ（仮説）、持続可能な AI 活用ルール、**外部可視化した成果物 1 本**

### 12-2. 自己診断チェックリスト（メモ２・３を統合）

#### 危険信号（当てはまるほど優先度高）

- [ ] 現在の仕事の **60% 以上** が「中間業務」（調査・資料作成・定型分析等）だ  
- [ ] AI を「完成品を作るためのツール」としてだけ使っている  
- [ ] 「AI のニュースは追っているが、実務に活かしていない」  
- [ ] 自分の業務の「AI に渡せる形での言語化」ができていない  
- [ ] 「自分がずっと考えられるテーマ」を**仕事文脈で** 3 つ即座に言えない  
- [ ] 組織内で「自分がどこに効いているか」が曖昧だ  
- [ ] （メモ３）AI に任せてみたことがほとんどない（理由が「怖い」「分からない」のまま放置）  
- [ ] （メモ３）転職活動をしたことがなく、**市場価値の外部検証**がない  
- [ ] （メモ３）今の会社の外でも同じことができると**自信を持って**言えない  
- [ ] （メモ３）変化のニュースが「楽しそう」ではなく「不安」としか感じない  

#### 好機のサイン（当てはまるほどチャンス）

- [ ] 特定のドメインに **5 年以上** の実務経験がある  
- [ ] 「なぜこれは良い／悪いか」を**言語化する習慣**がある  
- [ ] 人を動かした経験がある（規模問わず）  
- [ ] 変化を面白がれるマインドがある  
- [ ] 「やりたいこと」が今この瞬間にいくつか浮かぶ  
- [ ] AI を使って何かを「作った」経験が **直近 3 ヶ月**にある  
- [ ] （メモ２）「この人と 1 泊 2 日の旅行に行っても楽しめるか」という採用ナラティブに**鵜呑みにせず**、自分では「一緒に危機のプロジェクトを乗り切った同僚がいるか」を代替指標にしてもよい

### 12-3. 市場価値 3 軸（徳谷氏系：希少性・市場性・再現性）

**軸①：希少性（どれだけ珍しいスキルの組み合わせを持っているか）**

問い：「この能力の組み合わせを持つ人は、日本に何人いるか？」  
判断のヒント：固有名詞で声がかかるか。「誰でもいい」ではなく「あなたに頼みたい」と言われる領域があるか。

**軸②：市場性（その能力が社会から求められているか）**

問い：「5 年後も同じ需要があるか？ AI に代替されにくいか？」  
確認のヒント：転職サイトで自分の経験の求人数を見る。成長市場か縮小市場か。

**軸③：再現性（他の組織でも同じ価値を発揮できるか）**

問い：「この組織を離れても、同じことができるか？」  
確認のヒント：社外プロジェクト、副業、転職経験があるか。**社内ルール・社内人脈**に依存していないか。

**反論**

- 3 軸は**自己採点が盛れる**。エージェントや他者フィードバックで矯正する。  
- 「希少」は**需要がない希少**になりうる。  
- 転職サイトの求人数は**求人広告の偏り**があり、市場性の代理指標として過信しない。
