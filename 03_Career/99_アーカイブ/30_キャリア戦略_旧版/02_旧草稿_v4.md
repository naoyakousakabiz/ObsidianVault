# Part 1：現状認識（What is happening?）

## 1-1. 三つの断絶

AI時代の変化は「量的な延長」ではなく「質的な断絶」だ。三つの断絶を理解することが全ての出発点になる。

---

### 断絶①：AIの自律化（コパイロット → エージェント）

**2024年まで**：AIは「指示して動かすツール」だった。プロンプトを入力すれば動く、止まれば止まる。人間が「How（どうやるか）」を指示し続けることが前提だった。

**2026年現在**：AIエージェントは「目標を与えれば自律的に完結させる実行者」になった。計画・実行・エラー修正・品質チェックまでを人間の介入なしでループする。人間の役割は「What（何を達成するか）」の定義だけになった。

```
旧モデル：人間が指示 → AIが実行 → 人間が確認 → 人間が次の指示
新モデル：人間が目標設定 → AIが計画・実行・自己修正 → 人間が最終判断
```

**なぜ重要か**：「AIを使いこなす」という言説が陳腐化した。AIに指示を出せることは、もはや差別化にならない。問題は「何を目標として与えるか」を判断できるかどうかだ。

---

### 断絶②：競争軸の移動（能力 → 運用品質）

**旧競争軸**：どのモデルが一番賢いか（ベンチマーク競争）

**新競争軸**：企業導入の障壁を誰が潰すか（運用品質競争）

Anthropicが埋めているのは「賢さ」ではなく、長時間実行・権限制御・監査可能性・メモリ保持といった「PoCが本番に上がれない理由」だ。派手なデモより地味な部品が市場を制する。

**個人への含意**：「最新AIを知っている」ことより「AIを実務で機能させられる」ことの価値が圧倒的に高くなった。

---

### 断絶③：「ずっと考えられること」の希少化

LayerX代表・福島良典氏の言葉がこの断絶を最も鋭く表現している：

> 「AIとの壁打ちが無限にできる時代だからこそ、本当に好きなテーマをずっと考えられることが唯一の優位性になる。情報格差・開発コスト・専門知識の壁というブロッカーが全部外れた今、残るのは純粋な思考の持久力だけだ」

かつてのブロッカー：
- 情報格差（専門家だけが知っている）
- 開発コスト（エンジニアが必要）
- 専門知識の壁（資格・経験が必要）

これらがすべてAIによって解除された。残る差異は「そのテーマについてずっと考え続けられるか」だけになった。

## 1-3. 「死滅回遊」としてのAI時代

Goodpatch代表・土屋尚史氏の比喩が、この時代を最も鮮烈に描写している。

### 構造的な一致

呪術廻戦の「死滅回遊」では、昨日まで普通の人間だった者に突然「術式（固有能力）」が与えられ、歴戦の術師たちと同じフィールドに放り込まれる。

AI時代も同じ構造だ：
- **術式の強制付与**：Claude Codeを使えば、非エンジニアでもアプリが作れる
- **覚醒しただけでは勝てない**：「動くもの」と「人が使いたいもの」の間には分厚い壁がある
- **ゲームマスターは個人に興味がない**：プラットフォーマーが見ているのは、人間がAIをどう使うかの集合的データだ
- **ルールが毎週変わる**：新モデルが毎月出る。先月のベストプラクティスが今月の時代遅れになる

### 決定的な違い

しかし死滅回遊との最大の違いがある：**これはゼロサムゲームではない**。

今まで「開発コストが見合わない」と放置されていた小さな課題、ニッチすぎて誰もプロダクトにしなかった不便——それらが突然、個人でも解けるようになった。「強さ」のゲームではなく「目の付けどころ」のゲームだ。

**失敗コストもゼロに近い**。100個作って1個当たればいい。しかも作るコストはほぼゼロ。死滅回遊と違い、失敗しても死なない。

# Part 2：構造変化（Why is this happening?）

## 2-1. 市場構造：誰が勝ち、誰が消えるか

### 「薄いレイヤー」が消える理由

Shin氏（OpenWork CPO室）の分析：AIスタートアップの競争は「何となく便利なレイヤーを横に広く取る戦い」から「誰が企業の本番導入障壁を一つずつ潰すか」へ移動した。

薄い中間レイヤーが消える三つの理由：

**理由①：基盤モデル企業の優位**  
モデルそのものを持っているため、コンテキスト管理・推論制御・ツール呼び出し・メモリ・評価基盤を一体設計できる。後付けのミドルウェアでは勝てない。

**理由②：企業購買の信頼問題**  
大企業は「この会社は5年後も残っているか」「監査・説明責任に耐えられるか」「障害時に誰が責任を取るか」を見る。Anthropic・OpenAI・Googleはここで圧倒的に有利だ。

**理由③：ベンダー数削減の圧力**  
AI導入が進むほど、企業はベンダー数を減らしたくなる。「できるだけ上流のプレイヤーに寄せたい」という購買行動は自然だ。

### 価値が残る領域

基盤企業が全部食うわけではない。**汎用で浅い価値**が食われるのであって、以下は残る：

| 残る領域 | 理由 |
|---|---|
| 深いドメイン知識×AI（Vertical） | 基盤企業が深い業務実装まで全部できない |
| 対人・身体的接触（介護・医療・建設） | 物理空間での判断と手技 |
| 関係資本・信頼（M&A・ハイエンドコンサル） | 人脈・信頼・判断の三位一体 |
| 意思決定の責任を取る役割 | AIは責任を取れない |

---

## 2-2. 組織の再編：ピラミッドから砂時計へ

### 何が起きているか

AIエージェントは、中間管理職の典型的業務（情報集約・報告書作成・進捗管理）を最も得意とする。これにより、ピラミッド型組織の「中間層」が急速に薄くなる。

```
【従来】ピラミッド型            【これから】砂時計型
      ▲                              ▲
     ▲▲▲  経営層                   ▲▲▲  経営層
    ▲▲▲▲▲ 管理職（厚）            ▲ ▲  AIエージェント群（薄）
   ▲▲▲▲▲▲▲ 現場                 ▲▲▲▲▲ 現場（少数精鋭）
```

**実例**：
- Klarnaは約700人分の顧客対応をAIが代替。従業員5,000人→3,800人
- Shopifyは新規採用前に「なぜAIでできないか」の証明を義務化
- Anthropic内でも全チーム（法務・マーケ・デザイン含む）がClaude Codeを活用

### 中間管理職の二極化

全員が消えるわけではない。AI時代に「価値ある管理職」と「不要な管理職」は以下で分かれる：

| 価値ある管理職 | 不要になる管理職 |
|---|---|
| 業務を分解し、AIに渡せる形に設計できる | 情報を上下に「バケツリレー」するだけ |
| 判断ポイントを言語化できる | 合意形成に逃げ続ける |
| AIアウトプットを評価・批判できる | ハンコを押すだけ |
| 異常を検知し、例外処理できる | ルーティンを回すだけ |

### エージェンティック・チームという新単位

McKinseyが提唱：**2〜5人の人間が、50〜100の専門的AIエージェントを監督・指揮する体制**が次世代チームの基本形になる。

これらのチームが「ワークチャート」（人間から人間への指揮命令ではなく、エージェント同士がAPIで直接対話する構造）として結びつき、従来の組織図に取って代わる。

---

## 2-3. ハーネスという競争軸

福島良典氏（LayerX）の核心的な整理：顧客が欲しいのは「AIエージェント」単体ではなく、**「ハーネス＋エージェント＋決定論コード」のセット**——業務の完成品だ。

**料理の比喩**：AIエージェントがシェフ、ハーネスが調理場。どんなに腕のいいシェフも、調理場の設計が悪ければ良い料理は作れない。

### ハーネスが解決する問題

長時間タスクになるとAIエージェントが起こす典型的な問題：
- 終了条件の誤認：「もう十分」と途中で完了宣言してしまう
- 品質の自己過信：「完璧です」と言うが実際は崩れている
- 累積的なズレ：少しずつズレた仕事が積み重なる

これをハーネス設計で対処する：
- 終了条件の外部化（チェックボックス形式）
- 視覚的品質チェック（Playwrightなどで「目」を与える）
- 決定論的な通過点（Tool実行後に必ずリンターを走らせるなど）

### AI時代の業務提供3類型

| 類型 | 例 | 対象 |
|---|---|---|
| AIマネージドサービス型 | バクラク、Harvey、Ramp | 共通領域・非競争業務 |
| AIプラットフォーム＋FDE型 | Ai Workforce、Palantir | 内製エンジニア不足企業の競争領域 |
| 内製型 | 三井物産デジタル等 | 競争優位直結のワークフロー |

# Part 3：個人戦略（What should I do?）

## 3-1. 「入口と出口」フレームワーク

梶谷健人氏（POSTS代表）の分析：AIが進化するほど「中間の作業」が代替され、人間の価値は両端に集約される。

```
┌─────────────────────────────────────────────────────────┐
│                      業務の全体像                          │
├──────────┬────────────────────────────┬─────────────────┤
│  入口     │         中間（AIが担う）        │     出口        │
├──────────┼────────────────────────────┼─────────────────┤
│ 問いを立てる│ 資料作成                      │ A：レビュー・   │
│ 課題設定   │ コーディング                   │    品質管理     │
│ 起案・推進  │ 調査・リサーチ                 │                │
│           │ レポーティング                  │ B：人を動かす   │
│           │ 初稿生成                        │    組織を変える  │
│           │ 定型分析・情報集約              │    合意形成     │
│           │ 議事録・メール初稿              │                │
└──────────┴────────────────────────────┴─────────────────┘
    ↑                     ↑                        ↑
   残る                  消える                    残る
```

**重要な洞察**：中間領域で生き残れるのはトップ0.01%のみ。3年かけて習得した「片の技」は、AI時代に最も不利なポジションになる。習得時間と陳腐化速度のバランスが完全に崩れているからだ。

---

## 3-2. 三つのキャリアパス

### パス① 起業家・事業責任者型（入口＋出口B）

**何をするか**：問いを立てて課題を設定し、AIを使って実行し、アウトプットで人を動かす。

**どんな人に向くか**：「やりたいことが無限に出てくる」人。この欲望こそが最大の資産だ。

**なぜ強いか**：AIとの壁打ちが無限にできる今、構想力と熱中できるテーマを持つ人間に飛躍的な差がつく。グッドパッチ社長・土屋氏は2ヶ月でアプリを13個作り、1ヶ月で全社員にClaude Codeを展開した。「やりたいことがあるから時間が溶けていく」という状態が理想だ。

**必要能力**：課題設定力・自走力・プレゼン力・セールス力・人間力

**注意点**：AIをコア・スキルの「1枚目のカード」にしてはいけない。AIはあくまで掛け算の乗数だ。

---

### パス② 専門性×ディレクター型（出口A）

**何をするか**：深い専門知識を持ちながら、AIアウトプットの品質を管理し、方向を決める。

**どんな人に向くか**：特定ドメインに深い知識がある人。そのドメインの「鑑定眼」を持つ人。

**なぜ強いか**：深津貴之氏（note CXO）の核心的な指摘：「AIを使えない人の多くは実はディレクション能力がない自分を告白しているようなもの。与えられた条件でツールが最も輝く環境を作ることがディレクションの本質だ」。AIが大量のアウトプットを生成する時代、それを評価・選別・磨き上げる力が希少になる。

**必要能力**：特定領域の深い専門スキル・品質判断力・ディレクション力・批判的評価力

**注意点**：「スペシャリスト」に閉じると弱い。「自分のドメイン出身のディレクター」を目指すのが最強の形だ（例：デザイン出身のAIプロダクトディレクター）。

---

### パス③ 中間領域トップ0.01%（超例外的ルート）

**何をするか**：特定の中間スキル（コーディング・文章・デザイン等）でAIが到達できない水準に達する。

**注意点**：茨の道。このルートを「なんとなく」選ぼうとしている人の大半は、実は上の2つのパスに移行すべきだ。本当に向いているのは、そのスキルに「狂えるほど熱中できる」人だけだ。

---

## 3-3. スキル掛け算の設計法

深津貴之×梶谷健人の対話から抽出した実践的な設計フレームワーク。

### ステップ1：最初の2枚を市場原理で選ぶ

人材の価値は「希少性×需要」で決まる。最初の2枚のカードは感情ではなく市場原理で選ぶ。

```
100人に1人のスキルA × 100人に1人のスキルB = 1万人に1人
```

**良い組み合わせの条件**：
- 同じ人が持つことが珍しい組み合わせ（例：法律知識×AIエンジニアリング）
- 一方が他方の価値を10倍にする組み合わせ（例：特定業界の深い知識×AI活用）
- 転職せずとも「場所を変える」だけで希少性が生まれる組み合わせ

**悪い組み合わせの例**：  
「AIができます」だけ（コモディティ）  
「資料作成が得意」だけ（AIが代替）  
「AIエージェント構築できます」だけ（基盤企業に吸収される）

### ステップ2：フィニッシュパターンを設計する

梶谷氏の格闘ゲーム比喩：自分の「超必殺技」を決め、そこに持ち込むための「浮かし技」をサブスキルで取る。

**例（営業が強みの場合）**：
- フィニッシュパターン：「クロージング」
- サブスキル候補：その場でプロトタイプを作れる能力 / 相手の課題を即座に構造化する能力 / ライフプランまで設計できる深い傾聴力

### ステップ3：3枚目以降は情熱で選ぶ

最初の2枚が「生きるため」なら、3枚目以降は「生きがいのため」だ。ここで初めて「自分が心から好きなテーマ」を持ち込む。

**重要な原則**：35歳までに一度「ジャンプ」せよ——給料を下げてでも全く違う文脈の何かを始めることが、後の独自性につながる（PIVOT小野武彦氏）。

## 3-4. 自己配置力という新しい知性

LayerX Chief of Staff・櫻井氏の概念。チェスの比喩で理解する。

**旧来のマネジメント観**：「自分はプレイヤーで、チームメンバーは駒。盤面の外から操作する」

**新しい自己配置力**：「自分自身も盤面上の一つの駒として捉える。この局面では、自分がどう動くのが最適か」

### 三つの構成要素

**①自己位置づけ認知**  
今の自分がどのポジションにいて、どれだけの影響力を持っているかを把握する力。

**②盤面認知**  
組織・事業・イシューを俯瞰し、構造とボトルネック、相互作用を理解する力。

**③自己操作能力**  
「自分は出るべきか退くべきか」「何に時間を使うべきか」を判断して行動できる力。

**なぜ重要か**：どれだけ優秀でも、「そのスキルが今必要とされていない場所」では成果にならない。「自分はナイトだ！」と言い張っても、求められているのがビショップなら意味がない。環境や状況に応じて自分の立ち位置を変えられるか——これが成長を止めない人の共通点だ。

---

# Part 4：必要な能力（What do I need?）

## 4-1. 優先度HIGH：絶対に磨くべき3つ

### ① 問いを立てる力（最重要・最希少）

**なぜ最重要か**：AIは「解く」が得意だが「問う」は人間の領域だ。AIが進化するほど、「何を解くべきか」を定義する能力の希少性が高まる。

**具体的に何ができるべきか**：
- 「この業務のどこが本当のボトルネックか」を特定できる
- 「解くべき問題」と「解いても意味がない問題」を区別できる
- 曖昧な課題を構造化し、測定可能な問いに変換できる

**鍛え方**：毎日「なぜこれをやっているのか」を5回問い続ける。答えが出なければ、その業務は「問いを立てる」訓練の素材だ。

---

### ② オーケストレーション能力

**何か**：複数のAIエージェントを指揮し、ビジネスの成果に結びつける力。「このタスクをどのエージェントに、どんな文脈を与えて、どの順序で渡すか」を設計できる。

深津氏の定義：「予見・指示の分解力・各プレイヤーの強みの把握・連携の設計」——これはAIへのディレクションでも組織へのマネジメントでも本質的に同じ構造だ。

**CLAUDE.md的思考**：自分の業務ルールや文脈をAIに渡せる形に言語化する能力。これが組織の知的資産になる時代が来た。

**鍛え方**：今週の業務を「AIに渡せる形で書き出す」練習を毎日する。渡せない部分が「自分がやるべき仕事」だ。

---

### ③ 人間力と関係資本

**なぜ重要か**：信頼は人間同士のやり取りの累積であり、AIに代替できない。組織変革は「正論」では動かない。「なんかいいかも」という非合理的なモメンタムが組織を動かす（メルカリ・ハヤカワ五味氏）。

**採用現場の変化**：複数の経営者が同じことを言う——「今まではスキルや経験を重視していたが、今はとにかくいいやつを採りたい。人間として一緒に働いて楽しい人を採りたい」。ホワイトカラーの専門性がAIに代替されるほど、人間としての魅力の相対的価値が上がる。

**指標**：「この人と1泊2日の旅行に行っても楽しめるか」——これが採用の判断基準になりつつある。

---

## 4-2. 優先度MID：あると差がつく2つ

### ④ 批判的評価力・審美眼

**なぜ価値が上がるか**：昔「頭でっかちの評論家」と馬鹿にされていたスキルが、AI時代の最重要スキルになった。AIが大量のアウトプットを生成する今、「良いものと悪いものの違いを言語化できる力」が最終的な品質を決める。

深津氏：「つまらない動画と面白い動画の違いを判別し、なぜつまらないかを言語化し、面白くするには何が必要かを言語化できるなら、AIにお願いすれば割と解決する」。

**鍛え方**：好きな分野の優れた作品・サービス・文章を毎日1つ分析する。「なぜこれはいいのか」を言語化する習慣。

---

### ⑤ ドメイン深掘り

**なぜ価値が上がるか**：AIがあらゆる専門知識をオンデマンドで提供できる今、「AIと組み合わせることで誰も届けられない価値」を持つドメインエキスパートの希少性が上がる。

入山章栄教授（早大WBS）：「私がAIを使って生産性10倍以上になれているのは、経営学のドメイン知識を持ち、アウトプットの善し悪しを判断できるからだ。ドメイン知識なしにAIを使っても、ジャッジができない」。

**注意点**：ドメイン知識は「その分野の常識を全部知っている」ことではなく、「その分野特有のボトルネックと文脈を理解している」ことだ。

---

## 4-3. 優先度LOW：AIに委ねていいもの

以下はAIが既に十分な品質で代替できる。時間を使い続けるのは機会費用の損失だ。

- 情報収集・調査・サマリー作成
- コード初稿・ドキュメント初稿
- 定型分析・レポーティング
- 議事録・メール初稿
- スケジュール管理・調整
- 定型的な翻訳

**重要な心構え**：これらをAIに委ねることで生まれた時間を、優先度HIGH/MIDのスキルに全振りすること。「楽になった」で終わると、競争力は上がらない。

# Part 5：組織変革（How to transform organizations?）

## 5-1. 1%の設計者と99%の実行者

**なぜ「全員研修」が失敗するか**：日本企業の典型的失敗パターン——100人に研修→3ヶ月後に誰も使っていない→300万円のドブ捨て。

原因は構造にある。iPhoneは委員会で設計されなかった。1人のビジョナリーが「こういうものを作る」と決め、数千人が実行した。AI導入も同じ構造が成立する。

### 1%の設計者の条件

| 条件 | 詳細 |
|---|---|
| 業務全体の横断的理解 | 複数部門の実務経験。「点」ではなく「面」で見える |
| AIの可能性と限界の両方を知る | 技術詳細より「何が得意で何が苦手か」の実践的理解 |
| コスト感覚 | 「AIが勝つ部分」を数字で瞬時に判断できる |
| 即断力 | 合意形成に逃げず、「これはAI、これは人間」と即断できる |

**注意**：この人物がいない場合、外部から連れてくることを本気で検討すべきだ。副業でも週3日の契約でもいい。1%の設計者がいるかいないかで、会社のAI化の成否がほぼ決まる。

### 262の法則で考える

どの組織でも、AI活用状況は以下に分かれる：

```
上位2割：何も言われなくても積極活用
中間6割：様子を見ている（ここが勝負）
下位2割：どんなに言われても動かない
```

**正しいアプローチ**：下位2割を無理に動かそうとしない。上位2割の熱量と知見を中間6割に展開し、中間6割を積極層にコンバートする。結果として下位2割も空気で動く。

---

## 5-2. モメンタムの作り方

メルカリ AIストラテジー・ハヤカワ五味氏の実践から抽出した変革の4フェーズ。

### Phase 1：盛り下がりを先に潰す

「AIは万能」という過剰期待を先に下げる。苦手なことを正直に伝え、「ガッカリ」がモメンタムを殺す前に手を打つ。

**具体的に**：勉強会で「AIが苦手なことリスト」を共有する。期待値を下げることが、次のステップへの最大の準備になる。

### Phase 2：小さな火を守る

前向きな1〜2割が動ける環境を先に整備する。「使っていいツール」「確認不要の範囲」を明確化。「やりたいなら手を挙げて」ではなく、「この範囲ならノーチェックでやっていい」という環境を先に作る。

### Phase 3：社内PMFを狙う

ある時点でオセロが一気にひっくり返るように組織が変わる。そのために：
- 評価制度にAI活用度を組み込む
- 経営陣自身の「最初の不安と乗り越え」を語らせる（命令ではなく共感で）
- AI活用の成功者を可視化してMVPにする
- 「〇〇部署が成果出してる」という話題を意図的に流す

### Phase 4：経営のレールを引く

「AIを使わないと損、使えば評価される」というレールを経営側が引く。制度に組み込まれたら、あとは自然に広まる。

**社内PMFの証拠**：新規採用前に「なぜAIで代替できないか」の証明を求める（Shopify型）。AI予算を人件費と代替的に設計する（LayerX型）。

---

## 5-3. LayerXの実装例

福島良典氏（LayerX代表）の具体的な実践：

**「トークン消費量」をKPIにする**  
「トークンコスト＝人からAIに移った仕事量」と定義。これが劇的に増加している。

**ノーミーティングデーの設計**  
週1日はミーティングを入れない日として、役員含め全員でAIを触る時間にする。

**AI予算の再設計**  
「Slackのシート数×人数」という計算をやめ、まずAI予算を確保する。Claude / Cursor / その他AIツール、何に使ってもいい予算として人件費と代替的に設計。

**トップのコミットメント**  
「トップが権限委譲という言葉に逃げないことが最も大事。誰よりもAIを触ってコミットした上で、一部を委譲せよ」。

# Part 6：隠れたリスク（What could go wrong?）

## 6-1. 意思決定疲労（AI Brain Fry）

### 研究が示す衝撃的な事実

HBR（2026年2月）の研究：**「AIは仕事を減らさない。むしろ仕事を強化（Intensify）する」**

**メカニズム**：AIが処理を高速化する → 人間はより多くのプロジェクトを並走させられる → 次々とAIアウトプットを評価・承認・指示する「終わりのない意思決定ループ」に陥る → 研究者はこれを「AI Brain Fry（AIによる脳のショート）」と呼ぶ。

**実例**：Klarnaは急激なフラット化で残された従業員に深刻な判断疲労が生じたと報告されている。

### 対策

| 対策 | 具体的に |
|---|---|
| 「AIを止める時間」を設計する | 週1日はAIなしで考える時間を意図的に確保 |
| 全判断を自分が引き受けない仕組み | 「この種の判断はAIに委ねる」ルールを事前に作る |
| 判断の優先順位を明示化する | 「これは私が考える」「これはAIに任せる」を明確に |
| 余白を設計する | 人間同士が対話する時間をカレンダーに固定する |

---

## 6-2. AI迷子の増加

**現状**：オープンワーク大沢氏によると、2025年から求人検索でAIというキーワードが急増した。「何かしなければ」という漠然とした不安が満ちている。会社がアナログなのにメディアはAI一色——何をやったらいいか分からない「AI迷子」が激増している。

**AI迷子の特徴**：
- ニュースは毎日追うが何も変わらない（本屋で表紙を眺めるだけ）
- 研修を受けたが3ヶ月後に使っていない
- 「自分の仕事がなくなる」という不安と「何をすれば良いのか分からない」という迷いが同居している

**解決策**：AIを「最初の強み」にしない。既存のコアスキルに乗数としてAIを掛け算する設計をすること。

---

## 6-3. 受け身AI活用の危険性

**研究が示す問題**：AIに指示を出して待つだけの状態では、脳の活用領域が著しく低下する。受け身になった人間は「AIのコピーペ屋」になり、AIの質を判断できなくなる。

**グッドパッチ社内の観察**（土屋氏）：1ヶ月で200個以上のアプリが社内に生まれたが、クオリティは作った人によって全然違う。「ポン出しでとりあえず出した」アプリと「何回も修正を加えて自分のサービスに近づけた」アプリでは圧倒的な差がある。道具が一緒だからみんな同じことができるわけではない。

**必要な心構え**：AIを「受け身で使う道具」ではなく「能動的に指揮する部下」として扱うこと。「仕事を作れる人間」にならない限り、AIを使っても競争優位は生まれない。

---

# Part 7：日本市場の特殊性（Japan context）

## 7-1. JTCという選択肢の再評価

**逆説的な現実**（PIVOT・小野武彦氏）：AIによる雇用不安が高まるほど、雇用が安定しているJTCの相対的価値が上がる。

**なぜか**：
- 日本は雇用規制が強く、AIによる急激なレイオフが起きにくい
- 世界的な混乱の中で、日本の「共産主義的な安定」が相対的強みになる可能性
- プロ型キャリアの放物線（40〜50代で収入が下がる）問題を回避できる

**ただし選抜基準が変わる**：ピラミッドは維持されるが、中で選ばれる人が変わる。「間違えないインデックス型エリート」から「野生的なリーダー」へ。40代での大企業トップが当たり前になる時代が来る。

**JTCで生き残る条件**：
- 業務を分解してAIに渡せる能力
- 決断から逃げない姿勢
- 手を動かして実際にものを作れる
- 「どこでも働ける」という市場価値の確認（転職活動はしなくていいが、活動だけは続ける）

---

## 7-2. 転職市場の変化

### 大企業転職特需のラストイヤー

オープンワーク大沢氏：2023〜2025年に続いたDX系・エンジニア系の大企業求人は埋まりつつある。「大企業転職特需のラストイヤー」になる可能性が高い。

今開いている求人の特徴：
- AI/GX関連の新ポジション（時代に合わせた開き方）
- 第2新卒での特定職種採用（新卒で埋められなかったポジション）
- 同一業界内での移籍（同じ業界で油が乗った人材）

### 転職の2つの失敗パターン

**失敗①：裏切り転職**  
辞める会社に不義理をする転職。業界内のレピュテーションは長く残る。年収が上がれば上がるほど、採用企業は時間をかけてリファレンスを取る。

**失敗②：自分で決めなかった転職**  
親・配偶者・同僚の意見や圧力で決めた転職。矢印が自分に向かわないため、うまくいかない時に引きずり続ける。

---

## 7-3. 採用の変化

AI時代に採用側が重視する観点の変化：

| 観点 | 旧（〜2024） | 新（2025〜） |
|---|---|---|
| スキル | 特定技術の深い知識 | 変化に対応できる柔軟性 |
| 経験 | 同職種での実績 | ドメイン知識×AI活用の組み合わせ |
| 人物 | 会社のルールに従える | 一緒に働いて楽しい人間力 |
| AI | 使えると加点 | 使えて当然・使えないと減点 |
| コミット | 会社へのロイヤルティ | ミッション・ビジョンへの共感 |

**ショートカットチェック**：「この人は今の仕事が楽しいか」——楽しければ自然に改善したいと思い、AIも活用したくなる。楽しくない人は「コピーアンドペーストで終わる」仕事になる（グッドパッチ土屋氏）。

## まとめ：問うべき唯一の問い

**「何を学ぶか」の時代は終わった。**

情報格差・開発コスト・専門知識の壁——これらのブロッカーがすべて消えた今、残る差異は一つだけだ。

> **「自分は何のために、誰のために、ずっと考え続けられるか」**

AIは問いを答えに変える機械だ。だからこそ人間の仕事は「正しい問いを立てること」へ収束する。そしてその問いを立て続けるための「熱中できるテーマ」こそが、誰も奪えない競争優位になる。

死滅回遊に参加しながら、ゲームの設計者の意図に乗るかどうかを自分で決めること。与えられた術式（AI）を使いながら、「何を作り、何を目指すか」だけは自分で決めること。

それがこの時代を生き抜く唯一の戦略だ。
