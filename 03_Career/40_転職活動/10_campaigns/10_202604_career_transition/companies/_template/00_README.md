# `companies/_template` — 中身の説明

`_template` を**丸ごとコピー**して `companies/<スラッグ>/` を作る。  
**複製した企業フォルダでは、この `00_README.md` は消してよい**（テンプレ用の目次だから）。

**キャンペーン全体の地図:** `../../../../00_index.md`

---

## `10_company_info.md` — 企業情報

| 項目 | 内容 |
|------|------|
| **用途** | その社専用の**企業ノート**。メタ・志望動機・企業理解・研究・逆質問の素。 |
| **ここに書く** | 応募情報（台帳と同じ）、なぜこの会社か、事業・ポジションの理解、**面接ログ**（いつ・誰・雰囲気・出た論点のメモ）。 |
| **ここに書かない** | 汎用の30秒自己紹介や「どの社でも同じ」ストーリー全文 → 期フォルダの **`30_prep_common.md`** |
| **ペア** | 想定・本番のQ&Aは **`10_qa.md`**。ログに1行書き、細かいQ&Aはそちらへ。 |

---

## `10_qa.md` — 面接Q&A

| 項目 | 内容 |
|------|------|
| **用途** | **面接で聞かれそうなこと／聞かれたこと**の中心。暗記・直前見返し用。 |
| **ここに書く** | 想定質問と回答要点、正本（`10_プロフィール`）のどの見出しに根拠があるか、深掘り・追問のメモ、**面接後の「実際に出た質問」**の転記。 |
| **ルール** | 実績・数値は **`03_Career/10_プロフィール/20_work_history.md`** など正本にないことは書かない。 |
| **ペア** | 面接の流れ・所感のログは **`10_company_info.md`** の「面接ログ」。 |

---

## 足りなくなったら（その社フォルダに追加）

| 例 | いつ |
|----|------|
| `jd.md` | 求人原文の抜き書き・キーワード整理を別に残したいとき |
| `derived_履歴書.md` / `derived_職務経歴書.md` | その社向けに削った提出文を分けて持ちたいとき |
| オファー条件メモ | 内定〜承諾の条件整理が必要なとき |

汎用マスタ（`03_Career/20_マスタ/`）は**直接いじらない**。派生はこの企業フォルダ側で。

---

## 複製の手順（おさらい）

1. 親の **`10_applications.md`** に1行追加し、**スラッグ**＝フォルダ名を決める  
2. `_template` を `companies/<スラッグ>/` にコピー  
3. 複製先で **`00_README.md` を削除**（推奨）  
4. **`10_company_info.md`** と **`10_qa.md`** のプレースホルダを埋める
