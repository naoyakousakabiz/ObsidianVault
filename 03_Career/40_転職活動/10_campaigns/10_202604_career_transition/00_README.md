# このキャンペーン期フォルダの使い方

**1応募・1社の詳細**は `companies/<スラッグ>/`（`10_company_info.md`・`10_qa.md`）。  
この直下にある **`00`〜`30` の4ファイル**は、**全社横断**の台帳・振り返り・汎用トーク用です。

**全体の地図:** `../../00_index.md`  
**企業フォルダのテンプレ説明:** `companies/_template/00_README.md`

---

## 横断4ファイル（何をどこに書くか）

| ファイル | 役割 | 触るタイミング |
|----------|------|----------------|
| **`10_applications.md`** | **母台帳**（1応募＝1行）。**スラッグの正**＝ `companies/` のフォルダ名。 | 応募・連絡・ステータスが動くたびに**最初に更新**。 |
| **`00_pipeline.md`** | **目標**と**ファネル件数**、**週次振り返り**（仮説・来週の1手）。 | **週1**＋件数が大きく動いたとき。個別企業の長文は書かない。 |
| **`30_prep_common.md`** | **どの社でも使う**自己紹介・実績ストーリー・典型問の**回答骨子**・逆質問の汎用リスト。 | **面接前**に見返し・推敲。社名・企業固有情報は書かない。 |
| **`20_interviews.md`** | **複数社にまたがる**気づき（よく聞かれた論点、答え方の弱点、仮説メモ）。 | 面接**その都度**ではなく、**パターンが見えたとき**。各社の詳細は `10_company_info`／`10_qa`。 |

**流れの目安:** 応募発生 → **`10_applications`** に1行 → 同じスラッグで `companies/` を用意 → 企業ノート＆Q&A。週末 → **`00_pipeline`**。面接前 → **`30_prep_common`** と各社の **`10_qa`**。

---

## よくある迷い

- **「この社だけの話」** → `companies/<スラッグ>/10_company_info.md`（＋ Q&A は `10_qa.md`）。
- **「どの社でも同じ答え」** → `30_prep_common.md`。
- **「市場全体の長文・調査」** → `30_キャリア戦略/30_synthesis/`。採用した一行だけ戦略メモへ。
- **事実・数値の正本** → 常に `03_Career/10_プロフィール/`。

## 新しい期フォルダを作るとき

この **`00_README.md` ごと** `10_campaigns/<新スラッグ>/` にコピーし、見出しや中の「202604」などを差し替えると運用が揃います。
