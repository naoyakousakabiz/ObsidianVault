# Obsidian Vault Operating System Guide

このVaultは「AI時代のSecond Brain」として設計されています。

単なるメモ保管庫ではなく、

- 情報収集
- 知識化
- 行動管理
- 人生管理
- コンテンツ発信

を統合する「人生OS」として機能する構造です。

---

# 設計思想

このVaultは以下の3つの原則で設計されています。

## 1 情報フロー設計

情報は以下の流れで処理されます。

Input → Knowledge → Projects → Output

意味

Input  
外部情報の収集

Knowledge  
自分の理解として整理された知識

Projects  
実行・意思決定

Output  
発信・成果物


この流れにより

「情報収集だけして終わる」
状態を防ぎます。

---

## 2 人生管理OS

このVaultは以下の領域を管理します。

Work  
Finance  
Family  
Health  
Hobbies  

つまり

仕事
お金
家族
健康
人生の楽しみ

を全て管理する構造になっています。

---

## 3 AI時代の知識管理

AI時代では

情報量ではなく
「構造」が価値になります。

そのため

Input（情報）
Knowledge（理解）

を明確に分離しています。
