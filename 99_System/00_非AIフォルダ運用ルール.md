# 非AIフォルダ運用ルール（共通）

## 目的
- `04_AI` 以外のフォルダでも、AIが「格納先」と「命名」を一意に判定できる状態を作る。
- 既存資産を壊さず、入口だけを統一して運用コストを下げる。

## 入口ルール
- 各トップフォルダの入口は `00_索引.md` に統一する。
- 詳細ルールは既存 `README.md` / `CLAUDE.md` を正本として参照する。

## 格納先判定（非AI）
| 条件 | 格納先 |
|---|---|
| 本業実務・業務ナレッジ | `01_Work/` |
| 個人事業・副業実務 | `02_Business/` |
| 転職活動・キャリア戦略 | `03_Career/` |
| 生活インフラ（健康・家計・契約） | `05_Life/` |
| 子ども関連 | `06_Child/` |
| 所有物管理 | `07_Asset/` |
| 人間関係・ネットワーク | `08_Relationship/` |
| 運動・身体パフォーマンス | `09_Athlete/` |
| 文化・教養・趣味学習 | `10_Culture/` |
| 美容・アンチエイジング | `11_Beauty/` |
| 外部インプット受け皿 | `20_Input/` |
| 日次・音声ログ | `80_Journal/` |

## 命名ルール（非AI）
- 基本: `NN_内容.md`（設計・索引・運用ルール）
- 日付ログ: `YYYY-MM-DD.md`（Daily）
- イベント: `YYYYMMDD_内容.md`
- 処理ログ: `_processed_log.md`

## 運用ルール
- 新規ノートは「概要1〜3行 + 正本リンク」を基本とする。
- 一時メモは各フォルダの一時置き場へ入れ、週次で昇格判断する。
- `98_NotebookLM_Sync/` は編集しない。

## AI依頼テンプレ（1行）
`以下を非AI運用ルールで配置してください｜目的:〇〇｜状態:正本/未昇格/アーカイブ｜領域:Work/Business/Career/Life/Child/Asset/Relationship/Athlete/Culture/Beauty/Input/Journal｜種別:索引/ルール/手順/メモ/ログ`
